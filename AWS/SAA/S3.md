### 1. S3 기본 개념

S3는 AWS의 핵심 오브젝트 스토리지 서비스. 무한 확장, 수많은 AWS 서비스와 통합됨.

**버킷(Bucket) 특성:**

- 이름은 **전 세계 모든 계정에서 전역 고유** (globally unique)
- 버킷 자체는 **특정 리전에 생성** — S3가 글로벌 서비스처럼 보여도 실제로는 리전 기반
- 이름 규칙: 소문자·숫자·하이픈만 가능, 3~63자, 대문자·밑줄·IP 형태 불가, `xn--` 시작 불가, `-s3alias` 끝 불가

**객체(Object) 특성:**

- **키(Key)** = 전체 경로 (예: `my_folder/my_file.txt`)
- S3에는 실제 디렉토리 개념 없음 — 슬래시가 포함된 긴 키 이름일 뿐
- 최대 객체 크기: **5TB**
- 5GB 초과 업로드 시 → **멀티파트 업로드** 필수
- 객체에 메타데이터(키/값), 태그(최대 10개), 버전 ID 포함 가능

### 2. 보안 ⭐⭐

**3가지 권한 방식:**

| 방식                      | 설명                                   |
| ----------------------- | ------------------------------------ |
| **IAM Policy**          | 사용자/역할 기반. 특정 사용자가 S3에 접근 가능한 API 지정 |
| **Bucket Policy**       | JSON 기반. 버킷 전체에 적용. 크로스 계정 허용 가능     |
| **ACL (Object/Bucket)** | 세분화된 객체 단위 제어. 현재는 비활성화 권장           |

**접근 허용 조건**: IAM 권한 허용 **OR** 리소스 정책 허용 **AND** 명시적 Deny 없음

**Bucket Policy JSON 구조:**

- `Resources`: 버킷과 객체 ARN
- `Effect`: Allow / Deny
- `Actions`: 허용/거부할 API (예: `s3:GetObject`)
- `Principal`: 적용 대상 계정 또는 사용자

**Block Public Access 설정:** 기업 데이터 유출 방지용 추가 보안 레이어. 버킷 정책으로 공개 설정을 해도 이 옵션이 켜져 있으면 **절대 공개 안 됨**. 계정 수준에서도 설정 가능.

> **시험 TIP**: S3를 공개해야 하는데 403 오류가 나면 → Block Public Access 설정 확인

### 3. 정적 웹사이트 호스팅

S3로 정적 웹사이트 호스팅 가능. URL 형식:

- `http://버킷명.s3-website-리전.amazonaws.com`
- `http://버킷명.s3-website.리전.amazonaws.com`

403 오류 → 버킷 정책에서 **퍼블릭 읽기 허용** 여부 확인

---

### 4. Versioning (버전 관리) ⭐

- **버킷 수준**에서 활성화
- 같은 키로 덮어쓰면 버전 번호 증가 (1, 2, 3...)
- 삭제 시 실제 삭제가 아니라 **삭제 마커(Delete Marker)** 추가 → 이전 버전 복원 가능
- 버전 관리 활성화 전 파일은 버전 **"null"** 로 처리
- 버전 관리 **일시 중단해도 기존 버전은 삭제되지 않음**

---

### 5. Replication (복제) ⭐⭐

**전제 조건**: 소스 버킷과 대상 버킷 모두 **버전 관리 활성화** 필수. S3에 IAM 읽기/쓰기 권한 필요. 복제는 **비동기**로 진행.

|종류|설명|사용 사례|
|---|---|---|
|**CRR** (Cross-Region)|다른 리전으로 복제|법규 준수, 지연 시간 감소, 계정 간 복제|
|**SRR** (Same-Region)|같은 리전으로 복제|로그 통합, 운영/개발 환경 실시간 동기화|

**복제 주의 사항 — 시험 빈출:**

- 복제 활성화 **이후 생성된 객체만** 자동 복제 → 기존 객체는 **S3 Batch Replication** 사용
- 삭제 마커는 복제 여부 선택 가능 (optional)
- **버전 ID로 삭제한 경우 버전 ID는 복제되지 않음**
- **복제 체이닝 없음**: 버킷1 → 버킷2 → 버킷3 설정 시, 버킷1 객체는 버킷3으로 **자동 복제 안 됨**

### 6. S3 Storage Classes ⭐⭐⭐ (숫자까지 암기)

**내구성**: 모든 클래스 동일 — **99.999999999% (11 9's)**. 천만 개 저장 시 1만 년에 한 번 손실. **가용성**: 클래스마다 다름.

| 스토리지 클래스                       | 가용성    | 검색 속도                                         | 최소 보관    | 특징                    |
| ------------------------------ | ------ | --------------------------------------------- | -------- | --------------------- |
| **S3 Standard**                | 99.99% | 즉시                                            | 없음       | 자주 접근, 기본값            |
| **S3 Standard-IA (표준 간헐적 접근)** | 99.9%  | 즉시                                            | 30일      | 가끔 접근, 검색 비용 발생       |
| **S3 One Zone-IA(원존 간헐적 접근)**  | 99.5%  | 즉시                                            | 30일      | 단일 AZ, AZ 소멸 시 데이터 손실 |
| **Glacier Instant**            | 99.9%  | **밀리초**                                       | **90일**  | 분기별 접근, 즉시 복원 필요 백업   |
| **Glacier Flexible**           | 99.99% | Expedited 1~5분 / Standard 3~5시간 / Bulk 5~12시간 | **90일**  | 일반 아카이브               |
| **Glacier Deep Archive**       | 99.99% | Standard 12시간 / Bulk 48시간                     | **180일** | 장기 보관, 최저 비용          |
| **Intelligent-Tiering**        | 99.9%  | 계층에 따라 다름                                     | 없음       | 자동 계층 이동, 검색 비용 없음    |

**S3 Intelligent-Tiering 계층 이동 규칙:**

- Frequent Access: 기본 (자동)
- Infrequent Access: **30일** 미접근 시 (자동)
- Archive Instant: **90일** 미접근 시 (자동)
- Archive Access: 90~700일+ 설정 (선택)
- Deep Archive Access: 180~700일+ 설정 (선택)

### 시험 TIP 최종 정리

- S3 버킷 이름은 **전역 고유**, 하지만 버킷 자체는 **리전에 존재**
- **5TB** 최대 객체 크기, 5GB 초과 업로드 → **멀티파트 업로드**
- 퍼블릭 액세스 안 될 때 → **Block Public Access** 체크
- 복제 활성화 전 객체 복제 → **S3 Batch Replication**
- **체이닝 복제 없음** (1→2→3 설정해도 1→3 안 됨)
- Glacier 최소 보관: Instant/Flexible = **90일**, Deep Archive = **180일**
- One Zone-IA = AZ 하나만 사용 → **AZ 장애 시 데이터 손실 가능**
- Intelligent-Tiering = **검색 비용 없음**, 모니터링 비용만 발생
