## 문제 247

한 회사가 Amazon RDS for MySQL에 데이터베이스를 배포했습니다. 트랜잭션 증가로 인해 데이터베이스 지원 팀은 DB 인스턴스에 대한 읽기 속도가 느려지고 있다고 보고하며 읽기 전용 복제본(read replica)을 추가할 것을 권장하고 있습니다. 이러한 변경을 구현하기 전에 솔루션 아키텍트가 취해야 할 작업의 조합은 무엇입니까? (2개 선택)

- **A.** RDS 기본(primary) 노드에서 binlog 복제를 활성화합니다.
    
- **B.** 원본 DB 인스턴스의 장애 조치 우선순위(failover priority)를 선택합니다.
    
- **C.** 장기 실행 트랜잭션이 원본 DB 인스턴스에서 완료되도록 허용합니다.
    
- **D.** 글로벌 테이블을 생성하고 테이블을 사용할 수 있는 AWS 리전을 지정합니다.
    
- **E.** 백업 보존 기간을 0이 아닌 값으로 설정하여 원본 인스턴스에서 자동 백업(automatic backups)을 활성화합니다.

>[!.questions]- 정답
>정답 : C,E
> 읽기 전용 복제본을 생성하기 전에 해야할 필수 작업
> 1. 바이너리 로그(Binary Log) 활성화
> 	- RDS에서 자동 백업을 활성화하면 자동으로 바이너리 로그 활성화됨
> 2. 스냅샷 주의사항
> 	- 읽기 전용 복제본을 생성할 때 RDS는 원본의 스냅샷을 찍음. 만약 원본 DB에서 긴 트랜잭션이 실행 중이면, 스냅샷 과정이 지연되므로 미리 완료시켜야함
