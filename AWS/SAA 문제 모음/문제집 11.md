## 문제 101

솔루션 아키텍트가 퍼블릭 및 프라이빗 서브넷이 있는 VPC를 설계하고 있습니다. VPC와 서브넷은 IPv4 CIDR 블록을 사용합니다. 고가용성을 위해 3개의 가용 영역(AZ) 각각에 퍼블릭 서브넷 1개와 프라이빗 서브넷 1개가 있습니다. 퍼블릭 서브넷에 인터넷 액세스를 제공하기 위해 인터넷 게이트웨이가 사용됩니다. 프라이빗 서브넷의 Amazon EC2 인스턴스들이 소프트웨어 업데이트를 다운로드할 수 있도록 인터넷에 액세스해야 합니다. 프라이빗 서브넷에서 인터넷 액세스를 활성화하려면 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** 각 AZ의 퍼블릭 서브넷마다 하나씩, 총 3개의 NAT 게이트웨이를 생성합니다. 각 AZ에 대해 비 VPC 트래픽을 해당 AZ의 NAT 게이트웨이로 전달하는 프라이빗 라우팅 테이블을 생성합니다.
    
- **B.** 각 AZ의 프라이빗 서브넷마다 하나씩, 총 3개의 NAT 인스턴스를 생성합니다. 각 AZ에 대해 비 VPC 트래픽을 해당 AZ의 NAT 인스턴스로 전달하는 프라이빗 라우팅 테이블을 생성합니다.
    
- **C.** 프라이빗 서브넷 중 하나에 두 번째 인터넷 게이트웨이를 생성합니다. 비 VPC 트래픽을 프라이빗 인터넷 게이트웨이로 전달하도록 프라이빗 서브넷의 라우팅 테이블을 업데이트합니다.
    
- **D.** 퍼블릭 서브넷 중 하나에 외부 송신 전용(egress-only) 인터넷 게이트웨이를 생성합니다. 비 VPC 트래픽을 외부 송신 전용 인터넷 게이트웨이로 전달하도록 프라이빗 서브넷의 라우팅 테이블을 업데이트합니다.

>[!.questions]- 정답
>정답 : A
>NAT Gateway : Outbound는 허용하지만, Inbound는 차단하는 보안 장치

## 문제 102

한 회사가 온프레미스 데이터 센터를 AWS로 마이그레이션하려고 합니다. 이 데이터 센터에는 NFS 기반 파일 시스템에 데이터를 저장하는 SFTP 서버가 호스팅되어 있습니다. 서버에는 전송해야 할 데이터가 200GB 있습니다. 서버는 Amazon Elastic File System(Amazon EFS) 파일 시스템을 사용하는 Amazon EC2 인스턴스에 호스팅되어야 합니다. 이 작업을 자동화하기 위해 솔루션 아키텍트가 취해야 할 단계의 조합은 무엇입니까? (2개 선택)

- **A.** EFS 파일 시스템과 동일한 가용 영역에 EC2 인스턴스를 시작합니다.
    
- **B.** 온프레미스 데이터 센터에 AWS DataSync 에이전트를 설치합니다.
    
- **C.** 데이터 저장을 위해 EC2 인스턴스에 보조 Amazon Elastic Block Store(Amazon EBS) 볼륨을 생성합니다.
    
- **D.** 운영 체제 복사 명령을 수동으로 사용하여 EC2 인스턴스로 데이터를 푸시합니다.
    
- **E.** AWS DataSync를 사용하여 온프레미스 SFTP 서버에 적합한 위치 구성을 생성합니다.

>[!.questions]- 정답
>정답 조건 : 온프레미스에서 데이터를 자동화하여 마이그레이션하는 과정에 대한 설명
>
> B (에이전트 설치) : 온프레미스 환경에 DataSync 에이전트를 설치해야함.
> E (위치 구성) : DataSync에서 데이터를 가져올 소스 위치를 설정해야함. 


## 문제 104

솔루션 아키텍트는 웹사이트를 위한 고가용성 인프라를 설계해야 합니다. 웹사이트는 Amazon EC2 인스턴스에서 실행되는 Windows 웹 서버에 의해 구동됩니다. 솔루션 아키텍트는 수천 개의 IP 주소에서 발생하는 대규모 DDoS 공격을 완화할 수 있는 솔루션을 구현해야 합니다. 웹사이트에서 다운타임은 허용되지 않습니다. 이러한 공격으로부터 웹사이트를 보호하기 위해 솔루션 아키텍트가 취해야 할 작업은 무엇입니까? (2개 선택)

- **A.** AWS Shield Advanced를 사용하여 DDoS 공격을 차단합니다.
    
- **B.** 공격자를 자동으로 차단하도록 Amazon GuardDuty를 구성합니다.
    
- **C.** 정적 및 동적 콘텐츠 모두에 Amazon CloudFront를 사용하도록 웹사이트를 구성합니다.
    
- **D.** AWS Lambda 함수를 사용하여 공격자의 IP 주소를 VPC 네트워크 ACL에 자동으로 추가합니다.
    
- **E.** CPU 사용률 80%로 설정된 대상 추적 조정 정책(target tracking scaling policy)과 함께 Auto Scaling 그룹에서 EC2 스팟 인스턴스를 사용합니다.

>[!.questions]- 정답
>정답 : A,C
>A : AWS Shield Advanced를 사용하여 대규모 공격 시 방어 가능
>C : CloudFront는 전 세계 엣지 로케이션을 통해 콘텐츠를 배포하고, 공격 트래픽이 발생하면 EC2 오기 전에 분산된 엣지에서 흡수하고 차단


## 문제 105

한 회사가 새로운 서버리스 워크로드를 배포할 준비를 하고 있습니다. 솔루션 아키텍트는 최소 권한 원칙을 사용하여 AWS Lambda 함수를 실행하는 데 사용될 권한을 구성해야 합니다. Amazon EventBridge(Amazon CloudWatch Events) 규칙이 이 함수를 호출할 것입니다. 이 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** 작업(action)으로 lambda:InvokeFunction을 지정하고 보안 주체(principal)를 지정하여 함수에 실행 역할을 추가합니다.
    
- **B.** 작업으로 lambda:InvokeFunction을, 보안 주체로 Service: lambda.amazonaws.com을 지정하여 함수에 실행 역할을 추가합니다.
    
- **C.** 작업으로 lambda:*를, 보안 주체로 Service: events.amazonaws.com을 지정하여 함수에 리소스 기반 정책(resource-based policy)을 추가합니다.
    
- **D.** 작업으로 lambda:InvokeFunction을, 보안 주체로 Service: events.amazonaws.com을 지정하여 함수에 리소스 기반 정책을 추가합니다

>[!.questions]- 정답
>정답 : D
>Lambda 권한 설정
>- 실행 역할 : Lambda 함수가 실행될 때, 이 함수가 다른 AWS 리소스에 접근하기 위해 사용하는 권한
>- 리소스 기반 정책 : 다른 서비스나 계정이 이 Lambda 함수를 호출할 수 있도록 허용하는 권한
>여기서는 EventBridge가 Lambda를 호출하므로, EventBridge가 Lambda를 호출하는 것을 허용한다 라는 내용이 명시되어야함.
>
>최소 권한 원칙
>- Action : 함수를 실행하는 데 필요한 최소한의 권한은 `lambda:InvokeFunction`
>- Principal : 권한은 가질 주체는 EventBridge 서비스인 events.amazonaws.com

>[!.questions]- 문항별 풀이
>A & B : 실행 역할(IAM Role)은 Lambda가 S3에 파일을 쓰거나 로그를 남길 때 필요한 것.
>C : `lambda:*`는 모든 권한을 다 주는 것이므로 최소 권한 원칙에 어긋남

## 문제 106

한 회사가 기밀 데이터를 Amazon S3에 저장할 준비를 하고 있습니다. 규정 준수상의 이유로 데이터는 저장 시(at rest) 암호화되어야 합니다. 감사 목적으로 암호화 키 사용이 로깅되어야 합니다. 키는 매년 교체되어야 합니다. 이러한 요구 사항을 충족하며 운영상 가장 효율적인 솔루션은 무엇입니까?

- **A.** 고객 제공 키를 사용한 서버 측 암호화(SSE-C)
    
- **B.** Amazon S3 관리형 키를 사용한 서버 측 암호화(SSE-S3)
    
- **C.** 수동 교체가 설정된 AWS KMS 키를 사용한 서버 측 암호화(SSE-KMS)
    
- **D.** 자동 교체가 설정된 AWS KMS 키를 사용한 서버 측 암호화(SSE-KMS)

>[!.questions]- 정답
>정답 : D
>감사 목적의 로깅
>SSE-KMS : 키를 사용할 떄마다 CloudTrail 기록이 남음.
>SSE-S3 : S3 수준의 접근 로그는 남지만, 암호화 키 사용에 대한 전용 감사 로그로는 부족

## 문제 109

한 회사가 Amazon S3에 데이터를 저장해야 하며 데이터가 변경되는 것을 방지해야 합니다. 회사는 Amazon S3에 업로드된 새 객체가, 회사가 수정하기로 결정할 때까지 특정되지 않은 기간 동안 변경할 수 없는 상태로 유지되기를 원합니다. 회사의 AWS 계정 내 특정 사용자만 객체를 삭제할 수 있는 권한을 가질 수 있어야 합니다. 솔루션 아키텍트는 이러한 요구 사항을 충족하기 위해 무엇을 해야 합니까?

- **A.** S3 Glacier 볼트를 생성합니다. 객체에 WORM(write-once, read-many) 볼트 잠금(vault lock) 정책을 적용합니다.
    
- **B.** S3 객체 잠금(Object Lock)이 활성화된 S3 버킷을 생성합니다. 버전 관리를 활성화합니다. 보존 기간을 100년으로 설정합니다. 새 객체에 대한 S3 버킷의 기본 보존 모드로 거버넌스(governance) 모드를 사용합니다.
    
- **C.** S3 버킷을 생성합니다. 객체를 수정하는 모든 S3 API 이벤트를 추적하려면 AWS CloudTrail을 사용합니다. 알림을 받으면 회사가 보유한 백업 버전에서 수정된 객체를 복원합니다.
    
- **D.** S3 객체 잠금이 활성화된 S3 버킷을 생성합니다. 버전 관리를 활성화합니다. 객체에 법적 보존(legal hold)을 추가합니다. 객체를 삭제해야 하는 사용자의 IAM 정책에 s3:PutObjectLegalHold 권한을 추가합니다.

>[!.questions]- 정답
>정답 : D
>정답 조건 : 법적 보존 + 보존 기간
>회사의 요구사항
>- 특정되지 않은  기간동안 변경할 수 없어야 한다.
>	- 해결 : 법적 보존(Legal Hold)는 만료 날짜가 없어서 무기한으로 객체 보호
>- 계정 내 특정 사용자만 객체를 삭제할 수 있어야 한다.
>	- 해결 : Legal Hold 가 걸려있으면 아무도 삭제할 수 없으므로 Legal Hold를 해제하기 위해 IAM 정책에 추가해서 특정 사용자에게만 삭제 권한 부여

>[!.questions]- 문항별 풀이
>B : 100년이라는 보존 기간을 특정했기 때문에 오답. 특정되지 않은 기간이 필요
