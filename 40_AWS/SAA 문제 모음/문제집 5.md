## 문제 1 - Appflow

한 회사의 애플리케이션이 데이터 수집을 위해 여러 SaaS(Software-as-a-Service) 소스와 통합되어 있습니다. 이 회사는 Amazon EC2 인스턴스를 실행하여 데이터를 수신하고 분석을 위해 Amazon S3 버킷에 데이터를 업로드합니다. 데이터를 수신하고 업로드하는 동일한 EC2 인스턴스는 업로드가 완료되면 사용자에게 알림도 보냅니다. 회사는 애플리케이션 성능 저하를 인지했으며 성능을 최대한 향상시키고자 합니다. 최소한의 운영 오버헤드로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** EC2 인스턴스가 스케일 아웃할 수 있도록 Auto Scaling 그룹을 생성합니다. S3 버킷으로의 업로드가 완료되면 Amazon Simple Notification Service(Amazon SNS) 주제로 이벤트를 보내도록 S3 이벤트 알림을 구성합니다.
    
- **B.** Amazon AppFlow 플로우를 생성하여 각 SaaS 소스와 S3 버킷 간에 데이터를 전송합니다. S3 버킷으로의 업로드가 완료되면 Amazon Simple Notification Service(Amazon SNS) 주제로 이벤트를 보내도록 S3 이벤트 알림을 구성합니다.
    
- **C.** 각 SaaS 소스에 대해 출력 데이터를 보낼 Amazon EventBridge(Amazon CloudWatch Events) 규칙을 생성합니다. S3 버킷을 규칙의 대상으로 구성합니다. S3 버킷으로의 업로드가 완료되면 이벤트를 보낼 두 번째 EventBridge(CloudWatch Events) 규칙을 생성합니다. Amazon Simple Notification Service(Amazon SNS) 주제를 두 번째 규칙의 대상으로 구성합니다.
    
- **D.** EC2 인스턴스 대신 사용할 Docker 컨테이너를 생성합니다. 컨테이너화된 애플리케이션을 Amazon Elastic Container Service(Amazon ECS)에서 호스팅합니다. S3 버킷으로의 업로드가 완료되면 Amazon CloudWatch Container Insights를 구성하여 Amazon Simple Notification Service(Amazon SNS) 주제로 이벤트를 보냅니다.

>[!.questions]- 정답
>정답 : B
> Amazon AppFlow 
> 다양한 SaaS 애플리케이션과 AWS 서비스간에 데이터를 안전하게 전송할 수 있도록 해주는 관리형 통합 서비스. EC2 인스턴스 관리 필요없음

>[!.questions]- 문항별 풀이
> A : 스케일 아웃으로 성능 저하는 해결하지만, EC2 운영 오버헤드

## 문제 2 - NAT

한 회사가 단일 VPC의 Amazon EC2 인스턴스에서 고가용성 이미지 처리 애플리케이션을 실행하고 있습니다. EC2 인스턴스는 여러 가용 영역의 여러 서브넷 내에서 실행됩니다. EC2 인스턴스는 서로 통신하지 않습니다. 그러나 EC2 인스턴스는 단일 NAT 게이트웨이를 통해 Amazon S3에서 이미지를 다운로드하고 Amazon S3에 이미지를 업로드합니다. 회사는 데이터 전송 요금에 대해 우려하고 있습니다. 회사가 리전 데이터 전송 요금을 피할 수 있는 가장 비용 효율적인 방법은 무엇입니까?

- **A.** 각 가용 영역에서 NAT 게이트웨이를 시작합니다.
    
- **B.** NAT 게이트웨이를 NAT 인스턴스로 교체합니다.
    
- **C.** Amazon S3용 게이트웨이 VPC 엔드포인트를 배포합니다.
    
- **D.** EC2 인스턴스를 실행할 EC2 Dedicated Host를 프로비저닝합니다.

>[!.questions]- 정답
>정답 : C
>NAT Gateway
>- 특정 가용 영역(AZ)에 속함. 
>- 시간당 요금, 데이터 처리 요금 발생
>S3 게이트웨이 VPC 엔드포인트
> : VPC 내부의 라우팅 테이블에 S3로 향하는 경로를 추가하여, 퍼블릭 인터넷이나 NAT를 거치지 않고 AWS 비공개 네트워크를 통해 S3에 직접 접속.

>[!.questions]- 문항별 풀이
>A : 각 AZ에 NAT을 두면, 리전 간 데이터 전송 요금은 줄이지만, 데이터 처리 요금은 지불해야함.
>B : NAT 인스턴스는 관리 오버헤드 증가하고, 데이터 전송 요금 못줄임
>D : EC2 Dedicated Host 는 하드웨어 점유권을 갖는 것으로 라이선스 규정 및 보안 요구사항일 떄 사용

## 문제 3 

한 회사에 Amazon S3에 백업되는 대량의 시간에 민감한 데이터를 생성하는 온프레미스 애플리케이션이 있습니다. 애플리케이션이 커짐에 따라 인터넷 대역폭 제한에 대한 사용자의 불만이 접수되고 있습니다. 솔루션 아키텍트는 Amazon S3에 대한 적시 백업을 허용하면서 내부 사용자의 인터넷 연결에 미치는 영향을 최소화하는 장기적인 솔루션을 설계해야 합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** AWS VPN 연결을 설정하고 VPC 게이트웨이 엔드포인트를 통해 모든 트래픽을 프록시합니다.
    
- **B.** 새로운 AWS Direct Connect 연결을 설정하고 이 새로운 연결을 통해 백업 트래픽을 보냅니다.
    
- **C.** 매일 AWS Snowball 디바이스를 주문합니다. Snowball 디바이스에 데이터를 로드하고 매일 디바이스를 AWS로 반환합니다.
    
- **D.** AWS Management Console을 통해 지원 티켓을 제출합니다. 계정에서 S3 서비스 제한 제거를 요청합니다.

>[!.questions]- 정답
>정답 : B 
>AWS Direct Connect : 인터넷 망을 거치지 않고, 고객의 데이터 센터와 AWS 사이에 전용 광케이블을 연결하여 보내느 서비스
>- 인터넷 최소화 : 인터넷 망을 쓰지 않으므로, 내부 사용자의 인터넷 속도에 영향X
>- 고성능 & 저지연 


## 문제 4 - S3

한 회사에 중요한 데이터가 포함된 Amazon S3 버킷이 있습니다. 회사는 실수로 데이터가 삭제되지 않도록 보호해야 합니다. 솔루션 아키텍트가 이러한 요구 사항을 충족하기 위해 취해야 하는 단계 조합은 무엇입니까? (2개 선택)

- **A.** S3 버킷에서 버전 관리를 활성화합니다.
    
- **B.** S3 버킷에서 MFA Delete를 활성화합니다.
    
- **C.** S3 버킷에 버킷 정책을 생성합니다.
    
- **D.** S3 버킷에서 기본 암호화를 활성화합니다.
    
- **E.** S3 버킷의 객체에 대한 수명 주기 정책을 생성합니다.
    
>[!.questions]- 정답
>정답 : A,B
>버전 관리 : 파일을 수정하거나 삭제할 때마다 상태 모두 저장
>MFA Delete : 객체의 버전을 영구적으로 삭제하거나, 버전 관리 설정을 변경하려고 할때 인증코드를 요구

>[!.questions]- 문항별 풀이
>C : 버킷 정책은 특정 유저의 접근을 막을 수는 있지만, 권한이 있는 사용자가 하는 짓을 막지는 못함
>D : 기본 암호화는 데이터가 유출됐을 떄, 읽지 못하게 하는 기능


## 문제 6 - Macie

한 회사에 상점에 마케팅 서비스를 제공하는 애플리케이션이 있습니다. 이 서비스는 상점 고객의 이전 구매를 기반으로 합니다. 상점은 SFTP를 통해 회사에 트랜잭션 데이터를 업로드하고, 데이터는 처리 및 분석되어 새로운 마케팅 제안을 생성합니다. 일부 파일은 크기가 200GB를 초과할 수 있습니다.

최근 회사는 일부 상점에서 포함되지 말았어야 할 개인 식별 정보(PII)가 포함된 파일을 업로드했음을 발견했습니다. 회사는 PII가 다시 공유될 경우 관리자에게 경고가 전송되기를 원합니다. 또한 회사는 자동화된 해결 조치를 원합니다. 최소한의 개발 노력으로 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** Amazon S3 버킷을 안전한 전송 지점으로 사용합니다. Amazon Inspector를 사용하여 버킷의 객체를 스캔합니다. 객체에 PII가 포함된 경우 S3 수명 주기 정책을 트리거하여 PII가 포함된 객체를 제거합니다.
    
- **B.** Amazon S3 버킷을 안전한 전송 지점으로 사용합니다. Amazon Macie를 사용하여 버킷의 객체를 스캔합니다. 객체에 PII가 포함된 경우 Amazon Simple Notification Service(Amazon SNS)를 사용하여 관리자에게 PII가 포함된 객체를 제거하라는 알림을 트리거합니다.
    
- **C.** AWS Lambda 함수에서 사용자 지정 스캔 알고리즘을 구현합니다. 버킷에 객체가 로드될 때 함수를 트리거합니다. 객체에 PII가 포함된 경우 Amazon Simple Notification Service(Amazon SNS)를 사용하여 관리자에게 PII가 포함된 객체를 제거하라는 알림을 트리거합니다.
    
- **D.** AWS Lambda 함수에서 사용자 지정 스캔 알고리즘을 구현합니다. 버킷에 객체가 로드될 때 함수를 트리거합니다. 객체에 PII가 포함된 경우 Amazon Simple Email Service(Amazon SES)를 사용하여 관리자에게 알림을 트리거하고 S3 수명 주기 정책을 트리거하여 PII가 포함된 객체를 제거합니다.

>[!.questions]- 정답
>정답 : B 
>Amazon Macie : 데이터를 기계 학습과 패턴 일치를 통해 자동으로 스캔하여 PII(개인 식별 정보)를 찾아내는 관리형 보안 서비스
>- 큰 용량의 데이터 효율적으로 스캔 가능


>[!.questions]- 문항 풀이
>A : Inspector는 EC2, ECR, Lambda 함수의 소프트웨어 취약점 스캔 도구

## 문제 7

- **A.** 필요한 리전을 지정하는 예약 인스턴스(Reserved Instances)를 구매합니다.
    
- **B.** 필요한 리전을 지정하는 온디맨드 용량 예약(On-Demand Capacity Reservation)을 생성합니다.
    
- **C.** 필요한 리전과 3개의 가용 영역을 지정하는 예약 인스턴스(Reserved Instances)를 구매합니다.
    
- **D.** 필요한 리전과 3개의 가용 영역을 지정하는 온디맨드 용량 예약(On-Demand Capacity Reservation)을 생성합니다.

여기서 말하는 EC2 용량은 컴퓨팅 자원을 말함.
ex) 서버의 vCPU, RAM, 인스턴스 슬롯 등

>[!.questions]- 정답
>On-Demand Capacity Reservation
>용량 보장 : 특정 가용 영역에서 인스턴스를 실행할 수 있는 컴퓨팅 자원을 확실하게 예약해줌.
>AZ 지정 필수 
>기간의 유연성 : 사용한 시간만큼만 요금 부과

>[!.questions]- 문항별 풀이
>A & C : 예약 인스턴스는 기본 1년 또는 3년 단위 목적으로 사용함
>B : 용량 예약은 특정 AZ의 물리적 자원을 선점하는 개념이므로, 리전 단위 설정은 개념적으로 틀린 말임

## 문제 8

한 회사의 웹 사이트에서 항목 카탈로그에 Amazon EC2 인스턴스 스토어를 사용합니다. 회사는 이 카탈로그의 고가용성을 보장하고 안전하고 내구성이 뛰어난 위치에 저장되기를 원합니다. 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** 카탈로그를 Amazon ElastiCache for Redis로 이동합니다.
    
- **B.** 더 큰 인스턴스 스토어가 있는 더 큰 EC2 인스턴스를 배포합니다.
    
- **C.** 인스턴스 스토어에서 Amazon S3 Glacier Deep Archive로 카탈로그를 이동합니다.
    
- **D.** 카탈로그를 Amazon Elastic File System(Amazon EFS) 파일 시스템으로 이동합니다

>[!.questions]- 정답
>정답 : D
> EC2 인스턴스 스토어
> : 임시 저장소임. 인스턴스를 중지하거나 장애가 생기면 다 날라감
> EFS
> - 내구성 : 여러 가용 영역에 복제되어 저장.
> - 고가용성 : 여러 AZ에 있는 수많은 EC2 인스턴스가 동시에 파일을 읽고 쓸 수 있음
> - 용량이 알아서 늘어나고 줄어듬

## 문제 9

한 회사가 매월 통화 스크립트 파일을 저장합니다. 사용자는 통화 후 1년 이내에는 파일에 무작위로 액세스하지만, 1년 후에는 파일에 거의 액세스하지 않습니다. 회사는 사용자가 생성된 지 1년 미만인 파일을 최대한 빨리 쿼리하고 검색할 수 있도록 하여 솔루션을 최적화하고자 합니다. 오래된 파일을 검색할 때 지연이 발생하는 것은 허용됩니다. 가장 비용 효율적으로 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** 각 파일을 태그와 함께 Amazon S3 Glacier Instant Retrieval에 저장합니다. 태그를 쿼리하여 S3 Glacier Instant Retrieval에서 파일을 검색합니다.
    
- **B.** 개별 파일을 Amazon S3 Intelligent-Tiering에 저장합니다. S3 수명 주기 정책을 사용하여 1년 후 파일을 S3 Glacier Flexible Retrieval로 이동합니다. Amazon Athena를 사용하여 Amazon S3에 있는 파일을 쿼리하고 검색합니다. S3 Glacier Select를 사용하여 S3 Glacier에 있는 파일을 쿼리하고 검색합니다.
    
- **C.** 개별 파일을 태그와 함께 Amazon S3 Standard 스토리지에 저장합니다. 각 아카이브에 대한 검색 메타데이터를 Amazon S3 Standard 스토리지에 저장합니다. S3 수명 주기 정책을 사용하여 1년 후 파일을 S3 Glacier Instant Retrieval로 이동합니다. Amazon S3에서 메타데이터를 검색하여 파일을 쿼리하고 검색합니다.
    
- **D.** 개별 파일을 Amazon S3 Standard 스토리지에 저장합니다. S3 수명 주기 정책을 사용하여 1년 후 파일을 S3 Glacier Deep Archive로 이동합니다. Amazon RDS에 검색 메타데이터를 저장합니다. Amazon RDS에서 파일을 쿼리합니다. S3 Glacier Deep Archive에서 파일을 검색합니다

>[!.questions]- 정답
>정답 : D
>정답 조건 : 가장 비용 효율적 + 지연 허용
>S3 Standard : 가장 빠르고 검색 비용 없음
>Glacier Deep Archive : 데이터를 꺼내는 데 12~48시간이 걸리지만, S3 스토리지 중 가장 저렴
>RDS 메타데이터 : 빠른 쿼리 가능

>[!.questions]- 문항별 풀이
> A & C : Glacier Instant Retrieval 는 속도는 빠르지만 개비싸고, S3 태그로 쿼리하는 것은 비효율적
> B : Intelligent-Tiering은 모니터링 및 자동화 비용 추가로 듬

## 문제 10

한 회사에 1,000대의 Amazon EC2 Linux 인스턴스에서 실행되는 프로덕션 워크로드가 있습니다. 이 워크로드는 타사 소프트웨어로 구동됩니다. 회사는 중대한 보안 취약점을 해결하기 위해 최대한 빨리 모든 EC2 인스턴스의 타사 소프트웨어를 패치해야 합니다. 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** AWS Lambda 함수를 생성하여 모든 EC2 인스턴스에 패치를 적용합니다.
    
- **B.** AWS Systems Manager Patch Manager를 구성하여 모든 EC2 인스턴스에 패치를 적용합니다.
    
- **C.** AWS Systems Manager 유지 관리 기간을 예약하여 모든 EC2 인스턴스에 패치를 적용합니다.
    
- **D.** AWS Systems Manager Run Command를 사용하여 모든 EC2 인스턴스에 패치를 적용하는 사용자 지정 명령을 실행합니다

>[!.questions]- 정답
>정답 : D
>정답 조건 : 대규모 인스턴스 + 타사 소프트웨어 + 최대한 빨리
>AWS Systems Manager Run Command
>- 즉각성 : 예약이나 대기 없이 명령을 내리면 바로 수행
>- 확정성 : 태그를 활용해 한꺼번에 명령 가능
>- 유연성 : 사용자 지정 명령으로 타사 OS 패치

>[!.questions]- 문항별 풀이
>A : Lambda로 하면, 일일이 1000대 인스턴스에 작업해야함
>B : Patch Manager 는 주로 운영체제의 보안 업데이트나 특정 앱을 자동화하는데 사용
