- **C.** AWS Organizations를 사용하여 VPC가 인터넷에 액세스하지 못하도록 방지하는 서비스 제어 정책(SCP)을 구성합니다. ap-northeast-3를 제외한 모든 AWS 리전에 대한 액세스를 거부합니다.
    
- **D.** 각 VPC의 네트워크 ACL에 대한 아웃바운드 규칙을 생성하여 0.0.0.0/0으로의 모든 트래픽을 거부합니다. 각 사용자에 대해 ap-northeast-3 이외의 AWS 리전 사용을 방지하는 IAM 정책을 생성합니다.
    
- **E.** AWS Config를 사용하여 인터넷 게이트웨이를 감지하고 경고하며 ap-northeast-3 외부에 배포된 새 리소스를 감지하고 경고하는 관리형 규칙을 활성화합니다.

>	- 리전을 비활성할 수는 있음

## 문제 168

보안 팀이 팀의 모든 AWS 계정에서 특정 서비스나 작업에 대한 액세스를 제한하고자 합니다. 모든 계정은 AWS Organizations의 대규모 조직에 속해 있습니다. 솔루션은 확장 가능해야 하며, 권한을 유지 관리할 수 있는 단일 지점이 있어야 합니다. 솔루션 아키텍트는 이를 달성하기 위해 무엇을 해야 합니까?

- **A.** 서비스 또는 작업에 대한 액세스를 제공하기 위해 ACL을 생성합니다.
    
- **B.** 계정을 허용하기 위해 보안 그룹을 생성하고 사용자 그룹에 연결합니다.
    
- **C.** 서비스나 작업에 대한 액세스를 거부하기 위해 각 계정에 교차 계정 역할(cross-account roles)을 생성합니다.
    
- **D.** 서비스나 작업에 대한 액세스를 거부하기 위해 루트 조직 단위(root organizational unit)에 서비스 제어 정책(SCP)을 생성합니다

>[!.questions]- 정답
>정답 : D
> Service Control Policy (서비스 제어 정책)
> SCP는 AWS Organizations의 조직 혹은 OU내의 모든 계정에 대해 사용할 수 있는 최대 권한을 정의한다. 특정 서비스나 특정 작업을 SCP에서 deny하면, 해당 계정의 관리자라 할지라도 작업을 수행할 수 없음

 >[!.questions]- 문항별 풀이
 >A : ACL(Access Control List)은 주로 S3 버킷이나 서브넷 레벨에서 네트워크나 리소스 접근을 제어할 떄 사용.
