## 문제 171

한 회사가 품목 가격을 기반으로 세금 계산 조회를 자동화하는 API를 사용자에게 제공합니다. 이 회사는 연휴 기간에만 조회 수가 증가하여 응답 시간이 느려지는 문제를 겪습니다. 솔루션 아키텍트는 확장 가능하고(scalable) 탄력적인(elastic) 솔루션을 설계해야 합니다. 이를 달성하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** Amazon EC2 인스턴스에서 호스팅되는 API를 제공합니다. API 요청이 이루어지면 EC2 인스턴스가 필요한 계산을 수행합니다.
    
- **B.** 품목 이름을 수락하는 Amazon API Gateway를 사용하여 REST API를 설계합니다. API Gateway는 세금 계산을 위해 품목 이름을 AWS Lambda로 전달합니다.
    
- **C.** 뒤에 두 개의 Amazon EC2 인스턴스가 있는 Application Load Balancer를 생성합니다. EC2 인스턴스는 수신된 품목 이름에 대한 세금을 계산합니다.
    
- **D.** Amazon EC2 인스턴스에서 호스팅되는 API와 연결되는 Amazon API Gateway를 사용하여 REST API를 설계합니다. API Gateway는 세금 계산을 위해 품목 이름을 수락하고 EC2 인스턴스로 전달합니다.

>[!.questions]- 정답
>정답 : B
>정답 조건 : 확장 가능 + 탄력적 + 가변적인 트래픽
>서버리스 아키텍처 : API Gateway, Lambda
>가변적 트래픽 : Lambda
>탄력성 : 사용한 만큼만 지불

## 문제 172

솔루션 아키텍트가 애플리케이션을 위한 새로운 Amazon CloudFront 배포(distribution)를 생성하고 있습니다. 사용자가 제출한 정보 중 일부는 민감한 정보입니다. 애플리케이션은 HTTPS를 사용하지만 또 다른 보안 계층이 필요합니다. 민감한 정보는 전체 애플리케이션 스택에서 보호되어야 하며, 해당 정보에 대한 액세스는 특정 애플리케이션으로 제한되어야 합니다. 솔루션 아키텍트는 어떤 조치를 취해야 합니까?

- **A.** CloudFront 미리 서명된 URL(signed URL)을 구성합니다.
    
- **B.** CloudFront 미리 서명된 쿠키(signed cookie)를 구성합니다.
    
- **C.** CloudFront 필드 수준 암호화 프로파일(field-level encryption profile)을 구성합니다.
    
- **D.** CloudFront를 구성하고 뷰어 프로토콜 정책(Viewer Protocol Policy)에 대해 원본 프로토콜 정책(Origin Protocol Policy) 설정을 HTTPS 전용으로 설정합니다.

>[!.questions]- 정답
>정답 : C
>문제 조건 : HTTPS 외에 추가적인 보안 계층 + 전체 애플리케이션 스택에서 데이터 보호
>
>CloudFront 필드 수준 암호화
>엔드 투 엔드 보안 : HTTPS는 전송 중인 데이터를 암호화하지만, 데이터가 웹서버나 로드밸러서에 도착하여 SSL/TLS가 종료되면 데이터는 일시적으로 복호화된 평문 상태가 된다. 필드 수준 암호화는 데이터가 이미 암호화되어 원본까지 전달된다.
>특정 애플리케이션만 액세스 : CloudFront에 공개 키를 업로드하여 데이터를 암호화하고, 실제 해당 데이터를 처리해야 하는 앱만 개인 키를 보유하게 해서 중간 시스템들의 열람을 차단한다.

>[!.questions]- 문항별 풀이
>A & B : 미리 서명된 URL, 쿠키는 특정 사용자만 콘텐츠를 다운로드하거나 접근할 수 있도록 제한하는 권한 제어 용도. 


## 문제 175

전자상거래 회사가 Amazon API Gateway와 AWS Lambda 함수를 사용하는 주문 처리 애플리케이션을 가지고 있습니다. 이 애플리케이션은 데이터를 Amazon Aurora PostgreSQL 데이터베이스에 저장합니다. 최근 판매 이벤트 중 고객 주문이 갑자기 급증했습니다. 일부 고객은 시간 초과(timeouts)를 경험했으며 애플리케이션은 해당 고객들의 주문을 처리하지 않았습니다. 솔루션 아키텍트는 수많은 열린 연결(open connections)로 인해 데이터베이스의 CPU 및 메모리 사용률이 높다는 사실을 확인했습니다. 솔루션 아키텍트는 애플리케이션을 가능한 한 적게 변경하면서 시간 초과 오류를 방지해야 합니다. 어떤 솔루션이 이러한 요구 사항을 충족합니까?

- **A.** Lambda 함수에 프로비저닝된 동시성(provisioned concurrency)을 구성합니다. 여러 AWS 리전의 글로벌 데이터베이스가 되도록 데이터베이스를 수정합니다.
    
- **B.** Amazon RDS Proxy를 사용하여 데이터베이스용 프록시를 생성합니다. 데이터베이스 엔드포인트 대신 RDS Proxy 엔드포인트를 사용하도록 Lambda 함수를 수정합니다.
    
- **C.** 다른 AWS 리전에 데이터베이스에 대한 읽기 전용 복제본을 생성합니다. API Gateway의 쿼리 문자열(query string) 파라미터를 사용하여 트래픽을 읽기 전용 복제본으로 라우팅합니다.
    
- **D.** AWS Database Migration Service(AWS DMS)를 사용하여 데이터를 Aurora PostgreSQL에서 Amazon DynamoDB로 마이그레이션합니다. DynamoDB 테이블을 사용하도록 Lambda 함수를 수정합니다.

>[!.questions]- 정답
>정답 : B
>Lambda 와 DB 연결의 한계  : Lambda는 트래픽이 급증하면 스케일 아웃하여 처리한다. 이와 비례해서 DB 커넥션 또한 Lambda의 개수만큼 생성되기 때문에 타임아웃이 발생하게 되는 것이다.
>해결 
>- RDS Proxy의 역할 : RDS Proxy는 Lambda와 데이터베이스 사이에 위치하여 커넥션 풀을 관리한다. 수많은 Lambda가 짧은 시간 동안 요청을 보내도, 프록시가 이 연결들을 묶어 실제 DB에는 최소한의 연결만 유지

>[!questions]- 문항별 풀이
>A : Lambda의 Provisioned Concurrency는 Lambda의 Cold Start 문제를 해결. 그리고 Global Database 구성은 읽기 분석이나 재해 분석용이며 쓰기 연결을 막지 못함

## 문제 179

솔루션 아키텍트는 애플리케이션이 Amazon RDS DB 인스턴에 액세스하는 데 사용하는 데이터베이스 사용자 이름과 비밀번호를 안전하게 저장해야 합니다. 데이터베이스에 액세스하는 애플리케이션은 Amazon EC2 인스턴스에서 실행됩니다. 솔루션 아키텍트는 AWS Systems Manager Parameter Store에 보안 파라미터(secure parameter)를 생성하고자 합니다. 솔루션 아키텍트는 이 요구 사항을 충족하기 위해 무엇을 해야 합니까?

- **A.** Parameter Store 파라미터에 대한 읽기 액세스 권한이 있는 IAM 역할을 생성합니다. 파라미터를 암호화하는 데 사용되는 AWS Key Management Service(AWS KMS) 키에 대한 복호화(Decrypt) 액세스를 허용합니다. 이 IAM 역할을 EC2 인스턴스에 할당합니다.
    
- **B.** Parameter Store 파라미터에 대한 읽기 액세스를 허용하는 IAM 정책을 생성합니다. 파라미터를 암호화하는 데 사용되는 AWS Key Management Service(AWS KMS) 키에 대한 복호화 액세스를 허용합니다. 이 IAM 정책을 EC2 인스턴스에 할당합니다.
    
- **C.** Parameter Store 파라미터와 EC2 인스턴스 간에 IAM 신뢰 관계(trust relationship)를 생성합니다. 신뢰 정책(trust policy)에서 Amazon RDS를 보안 주체(principal)로 지정합니다.
    
- **D.** DB 인스턴스와 EC2 인스턴스 간에 IAM 신뢰 관계를 생성합니다. 신뢰 정책에서 Systems Manager를 보안 주체로 지정합니다.

>[!.questions]- 정답
>정답 : A
>- IAM 역할 사용 : EC2 인스턴스에서 애플리케이션에 권한을 부여할 때는 IAM '역할'
>- 보안 파라미터 : 파라미터 스토어에서 보안 파라미터를 생성하면 데이터가 암호화되어 저장된다. 읽기 권한과 KMS 키에 대한 복호화 권한이 있어야 한다.

>[!.questions]- 문항별 풀이
>B : '정책'은 IAM 역할이나 사용자에게 연결하는 것. 인스턴스 등 서비스에는 '역할'을 부여해야함
>

## 문제 184

한 회사가 소프트웨어 엔지니어링에 사용하는 AWS 계정을 가지고 있습니다. 이 AWS 계정은 한 쌍의 AWS Direct Connect 연결을 통해 회사의 온프레미스 데이터 센터에 액세스할 수 있습니다. 모든 비 VPC 트래픽은 가상 프라이빗 게이트웨이(virtual private gateway)로 라우팅됩니다. 개발 팀이 최근 콘솔을 통해 AWS Lambda 함수를 생성했습니다. 개발 팀은 이 함수가 회사의 데이터 센터 내 프라이빗 서브넷에서 실행되는 데이터베이스에 액세스할 수 있도록 허용해야 합니다. 어떤 솔루션이 이러한 요구 사항을 충족합니까?

- **A.** 적절한 보안 그룹과 함께 VPC에서 실행되도록 Lambda 함수를 구성합니다.
    
- **B.** AWS에서 데이터 센터로의 VPN 연결을 설정합니다. Lambda 함수의 트래픽을 VPN을 통해 라우팅합니다.
    
- **C.** Lambda 함수가 Direct Connect를 통해 온프레미스 데이터 센터에 액세스할 수 있도록 VPC의 라우팅 테이블을 업데이트합니다.
    
- **D.** 탄력적 IP(Elastic IP) 주소를 생성합니다. 탄력적 네트워크 인터페이스(ENI) 없이 탄력적 IP 주소를 통해 트래픽을 보내도록 Lambda 함수를 구성합니다

>[!.questions]- 정답
>정답 조건 : VPC 외부에서 실행되는 Lambda 함수를 어떻게 사설망과 연결된 내부 리소스에 접근시킬 것인가?
>- VPC 연동 : Lambda는 기본적으로 앱 VPC 외부에 있으므로, 프라이빗 DB에 접근하려면 VPC 내부에 들어갈 수 있어야 한다.
>- ENI 생성 : Lambda를 VPC 내의 특정 서브넷에 배치하면, AWS는 해당 서브넷에 탄력적 네트워크 인터페이스(ENI)를 생성한다. 이 ENI를 통해 Lambda의 트래픽이 VPC 내부로 흐른다.
>- 기존 라우팅 활용 : 일단 트래픽이 VPC 내부로 들어오면, 이미 설정된 가상 프라이빗 게이트웨이(VGW)와 Direct Connect의 라우팅 경로를 그대로 타고 온프레미스 데이터 센터까지 도달한다.
>- 보안 그룹 : VPC 내에서 통신하므로, Lambda 함수에서 나가는 트래픽을 허용하는 보안 그룹 설정 필요
>-> 같은 VPC로 Lambda를 참여시킨다.

## 문제 190
