## 문제 1

한 회사가 REST API를 통해 검색할 수 있도록 주문 배송 통계를 제공하는 애플리케이션을 개발 중입니다. 이 회사는 매일 아침 배송 통계를 추출하여 읽기 쉬운 HTML 형식으로 데이터를 구성하고 동시에 여러 이메일 주소로 보고서를 전송하고자 합니다. 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트가 취해야 하는 단계 조합은 무엇입니까? (2개 선택)

- **A.** Amazon Kinesis Data Firehose로 데이터를 전송하도록 애플리케이션을 구성합니다.
    
- **B.** Amazon Simple Email Service(Amazon SES)를 사용하여 데이터를 포맷하고 이메일로 보고서를 보냅니다.
    
- **C.** AWS Glue 작업을 호출하여 애플리케이션 API에서 데이터를 쿼리하는 Amazon EventBridge(Amazon CloudWatch Events) 예약 이벤트를 생성합니다.
    
- **D.** AWS Lambda 함수를 호출하여 애플리케이션 API에서 데이터를 쿼리하는 Amazon EventBridge(Amazon CloudWatch Events) 예약 이벤트를 생성합니다.
    
- **E.** 애플리케이션 데이터를 Amazon S3에 저장합니다. 이메일로 보고서를 보내기 위해 Amazon Simple Notification Service(Amazon SNS) 주제를 S3 이벤트 대상으로 생성합니다.

>[!.questions]- 정답
>정답 조건 = 매일 아침(스케줄링) + 이메일
> EventBridge : 특정 시간에 이벤트를 발생시키는 스케줄링 가능
> Lambda : 람다 함수 내부에서 API를 호출하여 데이터를 가져오고, HTML 변환 로직 수행
> Simple Email Service : 프로그래밍 방식으로 대량으로 한번에 전송 가능

>[!.questions]- 문항별 풀이
>A : Kinesis Firehose : 대량의 데이터를 S3나 Redshift 로 실시간 전달할 떄 쓰는 서비스
>B : AWS Glue : ETL 작업에 특화
>

## 문제 2

한 회사가 온프레미스 애플리케이션을 AWS로 마이그레이션하고자 합니다. 이 애플리케이션은 수십 기가바이트에서 수백 테라바이트에 이르는 다양한 크기의 출력 파일을 생성합니다. 애플리케이션 데이터는 표준 파일 시스템 구조에 저장되어야 합니다. 회사는 자동으로 확장되고, 고가용성을 제공하며, 운영 오버헤드가 최소화되는 솔루션을 원합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** Amazon Elastic Container Service(Amazon ECS)에서 컨테이너로 실행되도록 애플리케이션을 마이그레이션합니다. 스토리지로 Amazon S3를 사용합니다.
    
- **B.** Amazon Elastic Kubernetes Service(Amazon EKS)에서 컨테이너로 실행되도록 애플리케이션을 마이그레이션합니다. 스토리지로 Amazon Elastic Block Store(Amazon EBS)를 사용합니다.
    
- **C.** Multi-AZ Auto Scaling 그룹의 Amazon EC2 인스턴스로 애플리케이션을 마이그레이션합니다. 스토리지로 Amazon Elastic File System(Amazon EFS)을 사용합니다.
    
- **D.** Multi-AZ Auto Scaling 그룹의 Amazon EC2 인스턴스로 애플리케이션을 마이그레이션합니다. 스토리지로 Amazon Elastic Block Store(Amazon EBS)를 사용합니다.

>[!.questions]- 정답
>정답 : C
>정답 조건 : 표준 파일 시스템 구조 + 자동 확장 + 고가용성 + 운영 오버헤드 최소화
>EFS
>- 온프레미스에서는 리눅스의 NFS나 윈도우의 SMB 같은 파일 시스템 구조에서 동작. EFS는 이를 지원함
>- 자동 확장 
>- 고가용성
>	- 리전 서비스. 
>	- 데이터가 하나의 리전 내 여러 가용영역에 걸쳐 중복 저장. 특정 가용 영역에 장애가 나도 다른 영역에서 접근 가능

>[!.questions]- 문항별 풀이
>B & D : EBS 는 특정 가용 영역에 종속되는 스토리지. Multi-AZ 환경에서 가용 영역이 다른 EC2에 연결할 수 없음.
>또한 EBS는 용량이 자동으로 늘어나지 않음

## 문제 3 - S3 정책

한 회사가 회계 기록을 Amazon S3에 저장해야 합니다. 레코드는 1년 동안 즉시 액세스할 수 있어야 하며, 그 후 추가로 9년 동안 아카이브되어야 합니다. 관리자 및 루트 사용자를 포함하여 회사의 어느 누구도 10년의 전체 기간 동안 레코드를 삭제할 수 없어야 합니다. 레코드는 최대의 복원력으로 저장되어야 합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** 전체 10년 동안 S3 Glacier에 레코드를 저장합니다. 액세스 제어 정책을 사용하여 10년 동안 레코드 삭제를 거부합니다.
    
- **B.** S3 Intelligent-Tiering을 사용하여 레코드를 저장합니다. IAM 정책을 사용하여 레코드 삭제를 거부합니다. 10년 후, 삭제를 허용하도록 IAM 정책을 변경합니다.
    
- **C.** S3 수명 주기 정책을 사용하여 1년 후 레코드를 S3 Standard에서 S3 Glacier Deep Archive로 전환합니다. 10년 동안 규정 준수(compliance) 모드에서 S3 객체 잠금(Object Lock)을 사용합니다.
    
- **D.** S3 수명 주기 정책을 사용하여 1년 후 레코드를 S3 Standard에서 S3 One Zone-Infrequent Access(S3 One Zone-IA)로 전환합니다. 10년 동안 거버넌스(governance) 모드에서 S3 객체 잠금(Object Lock)을 사용합니다

>[!.questions]- 정답
>정답 : C
> S3 객체 잠금 - 규정 준수 모드
> 규정 준수 모드 : 루트 사용자를 포함한 어떤 사용자도 정해진 보존 기간 동안 객체를 삭제하거나 수정할 수 없음

>[!.questions]- 문항별 풀이
>A : 엑세스 제어 정책은 관리자나 루트 사용자가 언제든지 수정하거나 삭제 가능
>B : 루트 사용자는 IAM Policy 제한을 받지 않음
>D: 거버넌스 모드는 특별한 권한이 있는 사용자는 잠금을 해제하거나 삭제할 수 있음

## 문제 4

한 회사가 AWS에서 여러 Windows 워크로드를 실행합니다. 회사의 직원은 두 대의 Amazon EC2 인스턴스에서 호스팅되는 Windows 파일 공유를 사용합니다. 파일 공유는 서로 데이터를 동기화하고 중복 복사본을 유지합니다. 회사는 사용자가 현재 파일에 액세스하는 방식을 보존하는 고가용성 및 내구성을 갖춘 스토리지 솔루션을 원합니다. 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** 모든 데이터를 Amazon S3로 마이그레이션합니다. 사용자가 파일에 액세스할 수 있도록 IAM 인증을 설정합니다.
    
- **B.** Amazon S3 파일 게이트웨이를 설정합니다. 기존 EC2 인스턴에 S3 파일 게이트웨이를 탑재합니다.
    
- **C.** 파일 공유 환경을 Multi-AZ 구성의 Amazon FSx for Windows File Server로 확장합니다. 모든 데이터를 FSx for Windows File Server로 마이그레이션합니다.
    
- **D.** 파일 공유 환경을 Multi-AZ 구성의 Amazon Elastic File System(Amazon EFS)으로 확장합니다. 모든 데이터를 Amazon EFS로 마이그레이션합니다

>[!.questions]- 정답
>정답 : C
>FSx for Windows File Server
>- Windows 운영체제 기반으로 기존의 방식을 그대로 적용하면서 Multi-AZ로 가용성 업

>[!.questions]- 문항별 풀이
>B : 이 방식은 하이브리드 환경(온프레미스에서 클라우드 연결)에 적합

## 문제 5

솔루션 아키텍트가 여러 서브넷을 포함하는 VPC 아키텍처를 개발 중입니다. 이 아키텍처는 Amazon EC2 인스턴스와 Amazon RDS DB 인스턴스를 사용하는 애플리케이션을 호스팅합니다. 아키텍처는 두 개의 가용 영역에 있는 6개의 서브넷으로 구성됩니다. 각 가용 영역에는 퍼블릭 서브넷, 프라이빗 서브넷 및 데이터베이스용 전용 서브넷이 포함됩니다. 프라이빗 서브넷에서 실행되는 EC2 인스턴스만 RDS 데이터베이스에 액세스할 수 있어야 합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** 퍼블릭 서브넷의 CIDR 블록에 대한 라우팅을 제외하는 새 라우팅 테이블을 생성합니다. 라우팅 테이블을 데이터베이스 서브넷과 연결합니다.
    
- **B.** 퍼블릭 서브넷의 인스턴스에 할당된 보안 그룹으로부터의 인바운드 트래픽을 거부하는 보안 그룹을 생성합니다. 보안 그룹을 DB 인스턴스에 연결합니다.
    
- **C.** 프라이빗 서브넷의 인스턴스에 할당된 보안 그룹으로부터의 인바운드 트래픽을 허용하는 보안 그룹을 생성합니다. 보안 그룹을 DB 인스턴스에 연결합니다.
    
- **D.** 퍼블릭 서브넷과 프라이빗 서브넷 간에 새 피어링 연결을 생성합니다. 프라이빗 서브넷과 데이터베이스 서브넷 간에 다른 피어링 연결을 생성합니다.

>[!.questions]- 문항별 풀이
> A (라우팅 테이블 수정) : 라우팅 테이블은 서브넷 간의 길을 안내하는 지도 역할. 특정 서브넷을 제외한다고해서 보안이 완성되지 않음
> B : 보안그룹은 거부 규칙을 만들 수 없음
> D : VPC 피어링은 서로 다른 두 개의 VPC를 연결할 때 쓰는 서비스
> 

## 문제 6 - Gateway

한 회사가 Amazon Route 53에 도메인 이름을 등록했습니다. 이 회사는 ca-central-1 리전의 Amazon API Gateway를 백엔드 마이크로서비스 API를 위한 퍼블릭 인터페이스로 사용합니다. 타사 서비스가 이러한 API를 안전하게 사용합니다. 회사는 타사 서비스가 HTTPS를 사용할 수 있도록 회사의 도메인 이름과 해당 인증서로 API Gateway URL을 설계하려고 합니다. 이러한 요구 사항을 충족하는 솔루션은 무엇입니까?

- **A.** API Gateway에 Name="Endpoint-URL" 및 Value="회사 도메인 이름"인 스테이지 변수를 생성하여 기본 URL을 덮어씁니다. 회사 도메인 이름과 연결된 퍼블릭 인증서를 AWS Certificate Manager(ACM)로 가져옵니다.
    
- **B.** 회사 도메인 이름으로 Route 53 DNS 레코드를 생성합니다. 별칭(alias) 레코드가 리전별 API Gateway 스테이지 엔드포인트를 가리키도록 합니다. 회사 도메인 이름과 연결된 퍼블릭 인증서를 us-east-1 리전의 AWS Certificate Manager(ACM)로 가져옵니다.
    
- **C.** 리전별 API Gateway 엔드포인트를 생성합니다. API Gateway 엔드포인트를 회사 도메인 이름과 연결합니다. 회사 도메인 이름과 연결된 퍼블릭 인증서를 동일한 리전의 AWS Certificate Manager(ACM)로 가져옵니다. 인증서를 API Gateway 엔드포인트에 연결합니다. 트래픽을 API Gateway 엔드포인트로 라우팅하도록 Route 53을 구성합니다.
    
- **D.** 리전별 API Gateway 엔드포인트를 생성합니다. API Gateway 엔드포인트를 회사 도메인 이름과 연결합니다. 회사 도메인 이름과 연결된 퍼블릭 인증서를 us-east-1 리전의 AWS Certificate Manager(ACM)로 가져옵니다. 인증서를 API Gateway API에 연결합니다. 회사 도메인 이름으로 Route 53 DNS 레코드를 생성합니다. A 레코드가 회사 도메인 이름을 가리키도록 합니다

>[!.questions]- 정답 
>정답 : C
>- Gateway는 현재 문제에서 리전에 있으므로, SSL 인증서는 동일한 리전에 있어야함(만약 게이트웨이가 CloudFront에 있으면 버지니아 북부에 인증서 있어야함)
>- 그리고 Route 53에서 해당 도메인 이름에 대한 별칭 레코드를 생성하여 게이트웨이의 호스트 이름을 가치키도록 설정

>[!.questions]- 문항별 풀이
> A : 스테이지 변수는 API 내부 로직에 쓰이는 변수임. 도메인 이름 자체를 바꾸거나 인증서를 입히는 것이 아님
>  B & D : 리전별 엔드포인트를 구성할 때는 인증서가 Gateway 와 같은 리전에 있어야 함. `us-east-1`(버지니아 북부)는 엣지 최적화 엔드포인트용

## 문제 7 

한 회사가 인기 있는 소셜 미디어 웹사이트를 운영하고 있습니다. 이 웹사이트는 사용자에게 이미지를 업로드하여 다른 사용자와 공유할 수 있는 기능을 제공합니다. 회사는 이미지에 부적절한 콘텐츠가 포함되지 않도록 보장하고자 합니다. 회사는 개발 노력을 최소화하는 솔루션이 필요합니다. 이러한 요구 사항을 충족하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** Amazon Comprehend를 사용하여 부적절한 콘텐츠를 감지합니다. 신뢰도가 낮은 예측에 대해서는 인적 검토를 사용합니다.
    
- **B.** Amazon Rekognition을 사용하여 부적절한 콘텐츠를 감지합니다. 신뢰도가 낮은 예측에 대해서는 인적 검토를 사용합니다.
    
- **C.** Amazon SageMaker를 사용하여 부적절한 콘텐츠를 감지합니다. Ground Truth를 사용하여 신뢰도가 낮은 예측에 레이블을 지정합니다.
    
- **D.** AWS Fargate를 사용하여 부적절한 콘텐츠를 감지하는 사용자 지정 기계 학습 모델을 배포합니다. Ground Truth를 사용하여 신뢰도가 낮은 예측에 레이블을 지정합니다

>[!.questions]- 정답
>정답 : B
>Amazon Rekognition
>: 이미지와 비디오를 분석하기 위한 완전 관리형 시각 분석 서비스

>[!.questions]- 문항별 풀이
> A : Amazon Comprehend는 텍스트를 분석하는 서비스(문서의 감정 분석, 키워드 추출에 사용)
>  B : Amazon SageMaker 는 기계 학습 모델을 직접 빌드, 학습, 배포하는 플랫폼(모델을 직접 설계하고 데이터를 학습시켜야함)
>   D : AWS Fargate + 사용자 지정 모델은 직접 모델 개발과 배포까지 다 해야 하므로 복잡

## 문제 9 - Kinesis

한 회사가 300개 이상의 글로벌 웹사이트와 애플리케이션을 호스팅하고 있습니다. 회사는 매일 30TB 이상의 클릭스트림 데이터를 분석할 플랫폼이 필요합니다. 클릭스트림 데이터를 전송하고 처리하기 위해 솔루션 아키텍트는 무엇을 해야 합니까?

- **A.** AWS Data Pipeline을 설계하여 데이터를 Amazon S3 버킷에 아카이브하고 Amazon EMR 클러스터를 데이터와 함께 실행하여 분석을 생성합니다.
    
- **B.** 데이터를 처리하고 분석에 사용할 수 있도록 Amazon Redshift용 Amazon S3 데이터 레이크로 전송할 Amazon EC2 인스턴스의 Auto Scaling 그룹을 생성합니다.
    
- **C.** 데이터를 Amazon CloudFront에 캐시합니다. 데이터를 Amazon S3 버킷에 저장합니다. S3 버킷에 객체가 추가되면 AWS Lambda 함수를 실행하여 분석을 위해 데이터를 처리합니다.
    
- **D.** Amazon Kinesis Data Streams에서 데이터를 수집합니다. Amazon Kinesis Data Firehose를 사용하여 Amazon S3 데이터 레이크로 데이터를 전송합니다. 분석을 위해 Amazon Redshift에 데이터를 로드합니다

>[!.questions]- 정답 
>정답 : D
> 클릭스트림 데이터 : 사용자가 클릭하는 모든 정보
> Kinesis Data Streams : 실시간 스트리밍 데이터 대량 수집용 서비스
> Kinesis Data Firehose : 데이터를 압축,변환 후 S3나 Redshift로 전달하는 서비스
> Redshift :  많은 데이터를 쿼리하고 통계 내기 위한 서비스

>[!.questions]- 문항별 풀이
> A : Data Pipeline 는 주로 정기적인 시간 간격으로 데이터를 이동시키거나 변환하는 배치 작업에 쓰임(스트리밍 데이터 X)

1. 실시간 데이터 수집 (Kinesis Data Streams)
2. 데이터 전달 (Kinesis Data Firehose)
3. 저장 및 분석 (원본 보관은 S3, 고속 분석은 Redshift)
