## 스토리지 및 마이그레이션 (Storage & Migration)

### Amazon S3 스토리지 클래스 및 관리
- 패턴을 예측할 수 없는 데이터 : **S3 Intelligent-Tiering**
	- 수명 주기 자동화, 모니터링 비용만 발생, 검색 요금 없음
- 처음엔 자주 쓰다가 나중엔 안 쓰지만 즉시 꺼내야할 때(한달에 한번 정도 엑세스할때) : 30일 후 **S3 Standard-IA**로 전환
	- 가용성 유지, 즉각적 검색 보장
- 수년 단위의 가장 저렴한 장기 보관 : **S3 Glacier Deep Archive**
	- 12~48시간 복원, 법적 보관 목적
- 1년에 1~2번 접근하지만 즉시 꺼내야할 때 : **S3 Glacier Instant Retrieval**
- 실수로 파일이 삭제되는 것 방지 : **버전 관리 + MFA Delete 동시 활성화**
	- 루트 사용자만 권한 제어 가능
- 누구도 특정 기간 파일을 삭제, 수정하지 못하게 잠금 : **S3 객체 잠금 규정 준수 모드**
- 사용자가 직접 S3로 접속하는 것을 막고 CloudFront로만 접근하는 방법 : **원본 엑세스 ID(OAI) 또는 OAC**를 가진 CloudFront만 접근하도록 S3 버킷 정책 구성
- 최고 수준의 I/O 성능이 필요하지만, 인스턴스 중지 시 데이터가 날아가도 무방한 임시 작업(캐시 등)일 때 : **EC2 인스턴스 스토어**

### 네트워크 스토리지 및 전송
- Linux 환경에서 여러 EC2 인스턴스가 동시에 문서를 공유하고 자동 확장되는 스토리지 : **EFS**
	- 리눅스 기반, Multi-AZ 고가용성
- 온프레미스의 데이터를 AWS로 연결할 때 : **Storage Gateway**
	- Windows 환경에서 여러 서버가 파일 공유 : **FSx for Windows File Server**
		- SMB 프로토콜 호환, Multi-AZ
	- 온프레미스 NFS or SMB 환경을 유지하면서 S3 버킷을 온프레미스 환경에서 저지연 로컬 캐싱 + 자동 S3 백업 : **S3 File Gateway**
	- S3 또는 EFS에 FTP로 접근할 때 : **Transfer Family**
- 초대용량(수십 TB~) 데이터를 인터넷 한계 없이 가장 빠르게 전송 : **Snowball Edge** 물리적 배송
- 온프레미스와 AWS 간 빠르고 안정적인 온라인 데이터(EFS,S3 등) 지속 동기화 : **DataSync**
- 대용량 데이터를 글로벌 엣지를 통해 초고속으로 S3에 직접 업로드해야 할 때 : 엣지 로케이션을 활용하는 **S3 Transfer Acceleration**과 대용량 파일을 병렬로 올리는 **멀티파트 업로드**를 결합하여 전송 효율 극대화
- 실시간 동기화를 유지하고, 서로 다른 데이터베이스 엔진간의 이동이 필요할 때 : **AWS Schema Conversion Tool(SCT) + Database Migration Service(DMS)**
- 사내 사용자들의 인터넷 속도 저하 없이 대용량 S3 백업 : **AWS Direct Connect**를 통해 인터넷망 우회


## 서버리스 및 디커플링 (Serverless & Decoupling)
### 메시지 및 비동기 처리(SQS / SNS / EventBridge)
- 시스템 과부하 시 트랜잭션 유실 방지 및 비동기 분리 : **SQS**로 대기열에 임시 저장 후 처리
- 데이터가 정확히 순서대로 처리 : **SQS FIFO 큐** 사용
- 하나의 이벤트를 수십 개의 마이크로서비스로 동시 전달 : **SNS + SQS (Fan-out)패턴**
- SQS에서 같은 메시지가 중복 처리되는 문제 해결 : 애플리케이션 처리 시간보다 **가시성 제한 시간**을 길게 연장
- 매일 아침에 특정 작업을 실행해야 할 때 : **EventBridge(스케줄링) + Lambda**

### 컴퓨팅
- 초당 수백만 건의 트래픽을 밀리초 지연으로 처리하며 관리 오버헤드 최소화 : **API Gateway + Lambda + DynamoDB** 서버리스 아키텍처
- S3에 이미지/문서가 업로드되자마자 즉시 가공(압축/포맷 변경)할 때 : **S3 PUT 이벤트로 Lambda 트리거**
- 24시간 내내 일정하게 돌아가는 운영 인스턴스 : **예약 인스턴스**
- 단기 부하 시 필요한 가변 용량 : **스팟 인스턴스**
- 업무 시간에만 켜두고 중지할 수 있는 개발/테스트 서버 : **온디맨드 인스턴스**
- 코드만 업로드하면 모든 배포 과정(로드밸런싱, 자동확장, 상태 모니터링)을 알아서 처리 : **Elastic Beanstalk**

## 데이터베이스 및 분석 (Database & Analytics)
### 관계형 데이터베이스 (RDS / Aurora)
- 읽기 요청이 매우 많고 예측할 수 없는 환경 : **Aurora Multi-AZ + Auto Scaling** 
	- CPU 등 부하에 따라 읽기 복제본 자동 조절
- 장애 발생 시 데이터 손실과 가동 중지 시간을 최소화 : **Aurora Multi-AZ 배포 + RDS 프록시**
- 스토리지 I/O 성능 부족으로 쓰기 병목이 발생할 때 : **프로비저닝된 IOPS SSD**로 스토리지 타입 변경
- 데이터베이스 정기 테스트 비용 절감 : **스냅샷 생성 후 DB 삭제하고, 필용할 때 스냅샷에서 복원**
- 암호화되지 않은 상태로 운영 중인 RDS 인스턴스를 암호화할 때 : 기존 DB 스냅샷 생성 -> 스냅샷 복사 시 암호화 활성화 옵션 선택 -> 암호화된 새 스냅샷으로 새로운 DB 인스턴스 복원
- 운영 중인 데이터베이스에 부하를 주지 않고 스테이징/테스트용 데이터베이스를 즉시 생성해야 할 때 : 실제 데이터를 복사하지 않고 스토리지 층에서 포인터만 생성하는 **Aurora 데이터베이스 복제** 사용
- DynamoDB나 RDS 등의 데이터를 장기 보관하고 중앙에서 백업 일정을 자동 관리해야 할 때 : **AWS Backup**
- 애플리케이션의 데이터베이스 연결 수가 너무 많아질 때 효율적으로 관리해야할 때 : **RDS Proxy**
- 빠른 리전 간 복제가 필요할 때 : **Aurora DB**
### NoSQL 및 데이터 분석 (DynamoDB / Kinesis / Athena)
- DynamoDB 손상 시 15분 내 RPO 복구가 필요할 때 : **특정 시점 복구(PITR) 활성화
	- 초 단위 백업 제공
- 아침엔 안 쓰고 저녁에 트래픽이 폭증하는 예측 불가 워크로드 : **DynamoDB 온디멘드 용량 모드**
- 수만 건의 실시간 거래 데이터를 수집, 가공하여 여러 앱에 동시 공유(운영 오버헤드 있음) : **Kinesis Data Strams(멀티 컨슈머 지원) + Lambda**
- 대규모 스트리밍 데이터를 관리 오버헤드 없이 S3나 Redshift로 수집, 저장, 변환도 가능 : **Kinesis Data Firehose**
- SQL을 사용하여 흘러가는 데이터를 실시간으로 필터링하거나 변환 : **Kinesis Data Analytics**
- S3에 저장된 JSON 로그를 서버 구축없이 즉시 SQL로 분석 : **Athena**
	- 서버리스 쿼리 서비스
- 데이터 시각화 대시보드 제공 : **QuickSight**
- 이미지/비디오 분석 : **Rekognition**
- 데이터 추출, 변환, 적재를 위한 관리형 서버리스 서비스 : **AWS Glue**

## 네트워크 콘텐츠 전송 (Networing & Content Delivery)

### VPC 및 트래픽 제어
- 동일 리전 내 EC2에서 S3로 접근 시 데이터 전송 요금 절감 및 보안 향상 : **S3용 게이트웨이 VPC 엔드포인트**
	- 무료, 내부망 전송
	- S3나 DynamoDB에 접근할때만 사용 가능
- 타사 가상 방화벽 어플라이언스로 모든 트래픽을 투명하게 검사 : **Gateway Load Balancer(GWLB)배포 (GENEVE 프로토콜 사용)
- VPC 내 앱 서버에 안전하게 접속하기 위한 배스천 호스트 보안 설정 : 배스천은 외부 IP에서만 22번 포트 허용, 앱 서버는 배스천의 프라이빗 IP에서만 허용
- IP 지정 차단 및 허용, SQL Injection, XSS 방어, 특정 국가 차단, DDoS 방어 : **WAF**
- 타사나 외부 공급자 VPC의 특정 서비스에만 퍼블릭 인터넷을 거치지 않고 접속할 때 : **AWS PrivateLink**를 사용하여 대상 서비스 엔드포인트에 연결
- SSL/TLS 암호화 및 복호화 연산으로 인해 EC2 웹 서버의 CPU 컴퓨팅 용량이 한계일 때 : **ALB에 ACM 인증서를 적용**하여 SSL종료 작업 위임
- 서로 다른 VPC 안에 있는 서비스들이 서로 안전하게 통신해야할 때(서로의 사설IP주소를 사용하여 통신) : **VPC 피어링(peering)**
- 온프레미스와 AWS VPC를 연결할 때 : **Site-to-Site VPN**
### CloudFront vs Global Accelerator
- 웹사이트의 정적 및 동적 데이터 전송 지연 시간 단축 : S3(정적)와 ALB(동적)를 오리진으로 하는 CloudFront 배포
- UDP/TCP 통신의 초저지연 및 글로벌 리전 간 자동 장애 조치 필요 : Global Accelerator
	- 고정 애니캐스트 IP 제공

## 보안, 관리 및 운영 (Security, Identity & Management)

### 보안 및 암호화 (KMS, Macie, Shield)
- 다중 리전 S3 버킷을 동일한 키로 암호화/복호화하여 운영 효율화 : **고객 관리형 다중 리전 KMS 키 사용**
- DB 비밀번호의 주기적인 자동 교체 및 오버헤드 감소 : **AWS Secrets Manager**
- S3에 저장된 데이터 중 개인 식별 정보(PII)를 자동으로 찾아 알림 : **Macie**
- VPC 전체 트래픽 검사 및 도메인 접속 차단 등 세밀한 인라인 필터링 : **Network Firewall**
- 서드파티 DNS를 사용하며 대규모 DDoS 방어가 필요할 때 : ELB에 **Shield Advanced**
- EC2나 컨테이너의 취약점이나 네트워크 노출을 스캔하는 용도 : **Inspector**
- 로그를 분석하여 계정 내의 비정상적인 활동을 찾아내는 탐지 서비스 : **Guard Duty**
- S3 객체를 만료 기간 없이 무기한 보존하며, 특별 사용자만 잠금 해제할 때 : **S3 객체 잠금 중 법적 보관 기능** 사용
- 여러 리전과 여러 계정에 흩어진 AWS WAF을 중앙에서 일괄적으로 배포하고 관리하고 싶을 때 : **AWS Firewall Manager**
- 

### 운영 자동화 (Systems Manager, Config)
- 수천 대의 EC2 인스턴스에 타사 소프트웨어 패치를 즉시 적용 : Systems Manager Run Command
	- 사용자 지정 명령 일괄 실행
- SSH 키 분실 위험 없이 안전하게 인스턴스에 원격 접속 : **Systems Manager Session Manager**
	- IAM 권한 기반, 포트 개방 불필요
- 모든 인스턴스와 DB의 태그가 올바른지 감지하고 추적 : **AWS Config** 규정 준수 규칙 사용
- AWS 계정이 없는 경영진에게 안전하게 모니터링 화면 공유 : **CloudWatch 대시보드** 이메일 공유
- 여러 지표에 대해서 알람을 울리도록 하기 위할 떄 : Amazon CloudWatch 경보
- 인스턴스 유형별로 EC2 비용이 왜 증각했는지 분석할 떄 : **Cost Explorer**
- 비용/사용량에 대한 임계값에 대한 알림을 보낼 때 : **AWS Budgets**
- 외부에서 가져온 인증서 만료 알림 구축 : 가져온 인증서는 ACM이 자동 갱신해 주지 않으므로, **AWS Config**를 통해 만료 여부를 확인하고, **EventBridge**로 이벤트를 감지한 뒤 알림 
- SaaS 타사 서비스와 AWS 서비스 간에 데이터를 주고 받을 때 운영 비용 효율적인 방법 : **Amazon AppFlow** 사용
- PDF 형식이나 스캔된 의료 문서에서 보호 대상 건강 정보를 정확하게 식별해야 할 때 : 텍스트와 양식을 추출하는 **Amazon Textract**와 의료 정보를 파악하는 **Amazon Comprehend Medical**
