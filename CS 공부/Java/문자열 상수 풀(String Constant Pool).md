문자열 상수 풀은 JVM 내에 **Heap 영역에 존재하는 문자열 저장소**이다.

자바의 기본 타입(byte, char, int, boolean 등)은 스택에 저장된다.
반면에, 참조 타입의 데이터는 힙에 저장된다. 
스택에 존재하는 변수는 힙의 데이터를 참조한다.

![](../../images/스크린샷%202026-01-21%2015.34.39.png)

문자열을 생성하는 두 가지 방법에 따라 풀의 활용 여부가 달라진다.
프로그램이 처음 시작할때는 문자열 상수 풀은 기본적으로 비어있다.

## 리터럴 방식

아래는 리터럴 방식으로 문자열을 생성한 것이다.

`String s = "최영민";`

1. 문자열을 리터럴로 생성하면 JVM 은 먼저 **문자열 상수 풀**을 확인한다.
2. 동일한 값이 풀에 있으면 **기존의 참조 주소**를 반환한다.
3. 풀에 **값이 없으면** 새로운 문자열 **객체를 생성해서 풀에 등록**한다.


## new 연산자 방식

`String s = new String("최영민");`

1. new 연산자를 사용하면 **풀의 존재 여부와 상관없이** 힙 영역에 매번 **새로운 객체를 생성**한다.

## intern() 메서드

실행시점(Runtime)에 문자열을 상수 풀에 강제로 넣고 싶을 때 사용하는 메서드이다.

1. `intern()` 메서드를 호출하면, 해당 문자열이 풀에 있는지 확인한다.
2. 있으면 그 참조값을 반환하고, 없으면 풀에 추가한 뒤 참조값을 반환한다.

## 메모리 위치

Java 7 이상부터 문자열 상수 풀은 Heap 영역에 위치한다.

Heap 영역에 위치하게 됨으로써 GC의 대상이 되어서 참조하지 않는 문자열을 제거해주어 메모리 관리 효율성이 좋아졌다.
