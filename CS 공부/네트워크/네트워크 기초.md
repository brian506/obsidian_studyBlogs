**처리량** : 단위 시간당 데이터 전송량 (bps)
**지연 시간 (Latency)** : 요청 후 응답까지 걸리는 왕복 시간 (RTT)

## 네트워크 토폴로지

네트워크 토폴로지 (network topology)는 노드으의 링크가 어떻게 배치되어 있는지에 대한 방식이자 연결 형태를 의미한다.

### [트리 토폴로지] - 계층형 

트리형태로 배치한 네트워크 구성
![](../../images/스크린샷%202026-01-23%2014.04.12.png)

**특징** : 노드의 추가, 삭제가 쉬움
**단점** : 특정 노드에 트래픽이 집중될 때 하위 노드에 영향을 끼침


### [버스 토폴로지]

중앙 통신 회선 하나에 여러 개의 노드가 연결되어 공유하는 구조
- 근거리 통신망 (**LAN**)에서 사용
![](../../images/스크린샷%202026-01-23%2014.06.39.png)

**특징** : 설치 비용이 적고 신뢰성이 우수하며 중앙 통신 회선에 노드 추가, 삭제가 쉬움.
**단점** : 스푸핑에 취약

#### 스푸핑이란?
: 공격자가 승인된 사용자인 것처럼 속요서 시스템에 접근하거나 데이터를 가로채는 공격 행위

### [스타 토폴로지]

중앙에 있는 노드에 모두 연결된 네트워크 구성
![](../../images/스크린샷%202026-01-23%2014.14.09.png)

**특징** : 노드 추가 및 에러를 탐지하기 쉽고 패킷의 충돌 발생 가능성이 적음.
	 어떤 노드에 장애가 발생해도 쉽게 에러 발견 가능
	 장애노드가 중앙노드만 아니면 다른 노드에 영향을 적게 줌
**단점** : 중앙노드에 장애 발생 시 네트워크 전체가 마비되며 설치 비용 증가

### [링형 토폴로지]

각각의 노드가 양 옆의 두 노드와 연결하여 전체적으로 고리처럼 하나의 연속된 길을 통해 통신하는 구성
![](../../images/스크린샷%202026-01-23%2014.18.56.png)

**특징** : 노드 수가 증가되어도 네트워크상의 손실이 거의 없고 충돌이 발생되는 가능성이 적고 노드의 고장 발견을 쉽게 할 수 있음
**단점** : 네트워크 구성 변경이 어렵고 회선에 장애가 발생하면 전체 네트워크에 영향을 끼침

### [메시 토폴로지] - 그물망형

![](../../images/스크린샷%202026-01-23%2014.20.35.png)

**특징** : 특정 단말자치에 장애가 발생해도 여러 개의 경로가 존재하므로 네트워크를 계속 사용 가능하고 트래픽 분산 처리도 가능
**단점** : 노드의 추가가 어렵고 구축 비용과 운용 비용이 고가

## 병목 현상

**원인** : 네트워크 대역폭 부족, 비효율적인 토폴로지 구성, 서버 리소스 한계
**해결책** : *토폴로지*를 확인해서 어디서 병목이 생기는지 경로 파악

## 네트워크 성능 분석 

애플리케이션 코드 상 문제가 없지만, 사용자가 서비스로부터 데이터를 가져오지 못하는 상황일 때, 네트워크 병목 현상을 의심해봐야함.

병목 현상의 주된 원인
- 네트워크 대역폭
- 네트워크 토폴로지
- 서버 CPU, 메모리 사용량
- 비효율적인 네트워크 구성

서버 응답이 안 올 경우 서버와의 통신 상황을 파악하기 위해 명령어로 상태를 알 수 있다.

### [ping]

ping(Packet INternet Grooper)은 **네트워크 상태를 확인하려는 대상 노드**를 향해 **일정 크기의 패킷을 전송**하는 명령어이다.
TCP/IP 프로토콜 중에 ICMP 프로토콜을 통해 동작한다.

알 수 있는 정보
- 패킷 수신 상태와 도달하기까지의 시간
- 네트워크 연결 여무

명령어 : `ping [도메인주소]`

### [netstat]

**접속되어 있는 서비스들의 네트워크 상태**를 표시하는데 사용된다.

알 수 있는 정보
- 네트워크 접속
- 라우팅 테이블
- 네트워크 프로토콜 등 

![](../../images/스크린샷%202026-01-23%2015.21.21.png)

### [nslookup]

도메인 이름이 **올바른 IP로 매핑되는지(DNS 문제) 확인**하는데 사용된다.
특정 도메인의 IP주소를 확인하기 위해 사용한다.

![](../../images/스크린샷%202026-01-23%2015.24.13.png)


### [traceroute]

목적지 노드까지 네트워크 경로를 확인할 때 사용하는 명령어이다.
목적지 노드까지 구간들 중 어느 구간에서 응답시간이 느려지는 지를 확인할 수 있다.

## 대역폭

네트워크에서 대역폭이란 **단위시간당 전송할 수 있는 데이터의 최대 용량**을 의미한다.

네트워크 속도가 대역폭값에 근접할 경우 대역폭을 늘리면 속도가 빨라질수도 있지만,
보통 데이터 처리량과 네트워크 성능, 속도에 더 큰 영향을 미친다.
대역폭은 주로 **용량**과 밀접한 관계가 있다.

**단위** : bps
**네트워크 처리량** 
- 처리량은 **단위시간당 실제로 처리되는 데이터의 양**을 나타내고 대역폭 용량을 초과할 수 없다.

## 대역폭이 낭비되는 경우?

JWT 토큰의 길이가 길어질수록 매 통신마다 큰 데이터를 주고 받아야 하므로 대역폭의 낭비가 심화될 수 있다.
