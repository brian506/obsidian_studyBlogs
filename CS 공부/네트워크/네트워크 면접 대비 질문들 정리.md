
## 웹 통신의 큰 흐름: https://www.google.com/ 을 접속할 때 일어나는 일

**큰 흐름** 
주소 해석(DNS) -> 연결 수립(TCP/TLS) -> 서버 처리(HTTP/WAS) -> 브라우저 렌더링

먼저 브라우저는 URL을 파싱하여 HTTP 요청 메시지를 생성한다. 도메인 이름을 IP주소로 변환하기 위해 브라우저 캐시, OS 캐시 및 Host 파일을 확인하고 없으면 DNS 서버 순으로 조회하여 IP 주소를 획득한다.
이후 대상 서버와 TCP 3-way Handshake로 연결을 수립하고, HTTPS 인 경우 TLS 핸드셰이크를 통해 보안 세션을 맺는다.
연결이 되면 OS의 프로토콜 스택이 요청 메시지를 패킷으로 만들고, LAN 어댑터가 이를 전기 신호로 변환하여 송출한다. 이 패킷은 스위치와 라우터를 거쳐 ISP의 인터넷 핵심망을 타고 목적지 네트워크로 흘러간다.
목적지의 방화벽과 로드밸런서를 통과해 웹 서버에 도착하면, 서버의 프로토콜 스택이 패킷을 복원해 애플리케이션에 전달한다. 서버가 로직 처리 후 응답을 반환하면, 브라우저는 이를 받아 HTML/CSS를 파싱하고 랜더링하여 사용자에게 화면을 보여준다.
