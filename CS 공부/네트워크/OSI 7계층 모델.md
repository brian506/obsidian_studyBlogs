네트워크 통신을 계층화해서 독립적인 역할 분담을 하고, 문제발생 시 원인 파악을 용이하게 하기 위해 개발한 표준 규격이다.

![](../../images/스크린샷%202026-01-25%2018.58.04.png)

## 1계층 : 물리 계층 (Physical layer)

- 전기적 신호가 나가는 물리적인 장비
- 물리 계층에서는 전송하려는 데이터가 무엇인지, 어떤 에러가 있는지는 신경쓰지 않고 데이터 송수신에만 신경쓴다.
- 데이터를 **전기적인 신호로 변환해서 주고 받는 기능**만 있다.

**전송 단위** : 비트(bit)
**장비** : 케이블, 허브

## 2계층 : 데이터 링크 계층 (Data Link layer)

- 물리 계층에서 받은 **정보의 전송 오류를 감지하는 역할**을 수행한다.
- 안전한 정보의 전달을 수행하도록 해준다.
- MAC주소를 가지고 통신한다.
- `Point to Point` 간 신뢰성 있는 전송을 보장하기 위한 계층으로 CRC 기반의 **오류제어와 흐름 제어가 필요**하다.

**흐름 제어** : 송신 측과 수신 측의 속도 차이 조정
**오류 제어** : 오류 검출과 회복
**순서 제어** : 프레임의 순서적 전송

**전송 단위** : 프레임 (Frame)
**장비** : 스위치, 이더넷
**프로토콜** : MAC, LAN
## 3계층 : 네트워크 계층 (Network layer)

- 경로(Route)와 주소(IP)를 정하고 패킷을 전달해준다.
- 목적지까지 가장 안전하고 빠르게 데이터를 보내는 기능을 담당한다.
	- **최적의 경로**를 설정해야 한다.
- 전송계층에서 전달 받은 정보를 적당한 크기로 쪼개고, 쪼개진 정보들에 각각 **헤더를 추가해서 패킷을 생성**
	- 각각의 헤더는 논리적 주소를 포함

**전송 단위** : 패킷
**프로토콜** : IP, ICMP 등
## 4계층 : 전송 계층 (Transport layer)

- 양 끝단의 사용자들 간의 **신뢰성 있는 데이터를 주고 받게** 해주는 역할을 한다.
- 네트워크 계층의 패킷 전송을 제어하는 역할을 한다.
	- 패킷 재전송, 에러 복구, 흐름 제어 등
- 실질적인 정보 전송과 연결 다리 역할
- 데이터 전송을 위해 포트번호가 사용된다.

**전송 단위** : 세그먼트 (Segment)
**프로토콜** : TCP, UDP

## 5계층 : 세션 계층 (Session layer)

- 포트번호를 기반으로 통신 세션을 구성하고 상호작용과 동기화를 제공한다.
- 연결 세션에서 에러 복구, 데이터 교환을 담당한다.

## 6계층 : 표현 계층 (Presentation layer)

- 송/수신측간 데이터 형식을 정의한다.
- 수신한 데이터를 **변환, 검색, 암호화, 압축 등**의 과정을 거쳐 올바른 방식으로 변환하낟.
- 인코딩이나 암호화 등이 표현계층에서 일어난다.

**프토토콜** : png, jpg
## 7계층 : 응용 계층 (Application layer)

- 사용자와 직접 상호작용하는 계층

**프로토콜** : HTTP, SMTP, FTP 



## [TCP - IP 4계층](TCP%20-%20IP%204계층.md) 과의 비교

네트워크 전송 시 데이터 표준을 정리한 것이 OSI 7계층이라면, 이 이론을 실제 사용하는 인터넷 표준이 TCP / IP 4계층이다.
![](../../images/스크린샷%202026-01-25%2020.52.31.png)

**차이점** :
- OSI 모델은 역할을 기반으로 각 계층을 구성하고, TCP / IP 모델은 프로토콜의 집합을 기반으로 구성된다.
- 전체적인 네트워크 통신의 표준 방식은 OSI 모델이고, 데이터 전송 방식은 TCP / IP 모델이다.
