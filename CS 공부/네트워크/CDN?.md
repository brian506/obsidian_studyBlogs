#네트워크 

CDN(Content Delivery Network)은 컨텐츠 전송 네트워크이며, 전 세계에 분산된 서버들을 통해 웹 콘텐츠(이미지,비디오 등)를 사용자에게 물리적으로 가까운 서버에서 빠르게 전송해주는 시스템이다.

## 동작 방식
전세계에 있는 사용자는 각 지역의 EC2 를 사용하게 된다. 
여기서 만약에 서울에 있는 사용자가 미국 CDN에 있는 숏폼 컨텐츠를 요청한다고 가정하자.
서울 CDN에는 해당 숏폼 컨텐츠가 없으므로 미국 CDN에 요청해서 컨텐츠를 가져오고, 서울 CDN 에 마찬가지로 캐싱해둔다.
이렇게 되면 다른 서울에 있는 사용자가 같은 숏폼을 요청했을 때 미국 CDN 에 요청하지 않고, 서울 CDN 에서 해당 숏폼을 컨텐츠 만료 시점까지 캐싱된 컨텐츠를 바로 준다.
자주 사용하는 컨텐츠에 한해서 캐싱되며, 자주 사용하지 않으면 주기적으로 삭제된다.

## CDN 캐싱 방식

### Static Caching
- Origin Server 에 있는 컨텐츠를 운영자가 미리 Cache Server 에 복사 해두어, 사용자가 요청 시 Origin Server 로 가지 않고 곧바로 Cache Server 에서 준다.

### Dynamic Caching
- 사용자가 요청 시에 컨텐츠가 없으면 Origin Server 에 있는 컨텐츠를 요청하여 컨텐츠를 준다.
