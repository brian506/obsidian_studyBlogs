**Q1. [Java Core] Call by Value와 참조형 변수** 자바에서는 메서드에 매개변수를 전달할 때 'Call by Reference'가 존재하지 않고 오직 **'Call by Value(값에 의한 호출)'**로만 동작한다고 합니다. 그렇다면 메서드에 **참조형(Reference Type) 변수**를 넘겨서 객체의 내부 값을 변경했을 때, 원본 객체의 값이 변경되는 현상은 자바 메모리 구조(Stack, Heap) 관점에서 어떻게 설명할 수 있나요?

**Q2. [JPA] N+1 문제와 연관관계** 게시물(Board)과 댓글(Comment)이 1:N 양방향 연관관계로 설정되어 있을 때, 게시물 리스트를 조회하면 **N+1 문제**가 발생할 수 있습니다. N+1 문제가 무엇인지 예시를 들어 설명해 주시고, 이를 방지하기 위해 실무에서 **자식 객체 -> 부모 객체로의 '단방향 연관관계'** 설정을 권장하는 이유를 설명해 주세요.

**Q3. [JVM / GC] Heap 영역의 분리와 GC 동작** 가비지 컬렉션(GC)을 효율적으로 수행하기 위해 JVM의 힙(Heap) 영역은 크게 **Young Generation**과 **Old Generation**으로 나뉩니다. 이렇게 영역을 분리하게 된 **2가지 전제 조건(가설)**은 무엇이며, 각 영역에서 발생하는 GC(Minor GC vs Major GC)의 특징과 차이점을 설명해 주세요.

**Q4. [Spring] 빈(Bean) 스코프와 스레드 안전성** 스프링 컨테이너에서 생성되는 빈(Bean)은 기본적으로 **싱글톤(Singleton) 스코프**를 가집니다. 싱글톤 스코프의 특징은 무엇이며, 여러 스레드가 동시에 접근하는 웹 애플리케이션 환경에서 싱글톤 빈을 설계할 때 **내부에 인스턴스 변수(상태값)를 저장하지 않도록 설계해야 하는 이유**는 무엇인가요?

**Q5. [Java Core] 불변 객체(Immutable)와 방어적 복사** String과 같은 **불변 객체(Immutable Object)**가 멀티스레드 환경에서 동기화 처리 없이도 안전한(Thread-safe) 이유는 무엇인가요? 또한, 어떤 클래스를 완벽한 불변 객체로 만들기 위해 생성자로 가변 객체(예: List)를 인자로 받을 때, 원본을 그대로 참조하지 않고 **'방어적 복사(Defensive Copy)'**를 수행해야 하는 이유를 설명해 주세요.
