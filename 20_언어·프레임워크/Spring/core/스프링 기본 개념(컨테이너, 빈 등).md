## 스프링 빈(Bean)

> 스프링 컨테이너가 생성하고, 관계를 설정하며 관리하는 자바 객체

- 개발자가 직접 new 로 객체를 생성하는 것이 아니라, 스프링이 제어권을 가져가서(IoC) 스스로 만들고 관리
- `@Component`, `@Configuration` 등으로 등록

## 제어의 역전(IoC) & 의존성 주입 (DI)

**IoC** : 객체의 생명주기와 의존성 관리의 권한이 개발자가 아닌 컨테이너(ApplicationContext)에 있는 것
**DI** : IoC를 구현하는 구체적인 방법
- 개발자가 생성하는 것이 아닌 컨테이너에서 주입받는 방식
**생성자 주입을 권장**
1. 불변성 : 한 번 주입된 객체는 변하지 않도록 final 키워드 사용
2. 누락 방지 : 컴파일 시점에 의존성 주입이 누락된 것 확인 가능
3. 순환 참조 방지 : app 구동 시점에 순환 참조 오류 캐치

## Application Context (스프링 컨테이너)

> 빈의 생성, 설정, 조립, 관리 등 빈의 전체 생명주기 관리


## 빈 스코프 (Bean Scope)

- **싱글톤(Singleton) - 기본값** : 컨테이너에 단 **하나의 객체**만 생성해서 공유해야함.
	- 여러 스레드가 공유하기 때문에 상태를 가지지 않도록(인스턴스 변수 값 저장 X)
- **프로토타입(Prototype)** : 빈을 요청할 때마다 새로운 객체 생성

## 빈의 생명주기

1. 스프링 컨테이너 생성
2. 스프링 빈 생성
3. 의존관계 주입
4. 초기화 콜백 : 빈이 사용할 준비가 됨
5. 사용
6. 소멸 전 콜백 : 컨테이너 종료 전 빈 정리 작업
7. 스프링 종료
