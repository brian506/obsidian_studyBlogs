### 1. 모니터링 사각지대 — 보호자 SSE 끊김
현재 Prometheus 룰은 **피보호자 → 서버 GPS 업로드** 경로만 감지함 (`POST /api/v1/location/gps`). 보호자 앱의 SSE 끊김은 룰 밖.
→ 개선: `sse_active_connections` 커스텀 Micrometer 메트릭 export, 보호자 단위 룰 추가.

### 2. SSE 가상 스레드 매핑하여 polling 처리
Redis Pub/Sub 이용?

### 3. AI 에이전트 LLM 비용 폭주 위험
cache miss 가 짧은 시간에 몰리면 LLM 호출 비용 폭증 가능.
→ 개선: Rate limit, Circuit breaker, 시간당 호출 한도.

### 4. SPOF — nginx 단일 EC2
nginx EC2 한 대가 죽으면 트래픽 전부 정지.
→ 개선: ASG + nginx 2대, 또는 NLB로 마이그레이션 (SSE는 NLB가 idle timeout 더 김).

### 6. 멀티 AZ 미적용 (App)
현재 App Service는 단일 AZ. RDS만 multi_az.
→ 개선: 두 AZ에 ECS Service 분산, Cloud Map이 라운드로빈.

### 7. 정량 지표 부재
SSE 동시 연결 한계, MTTR(AI 도입 전/후), 비용 절감 수치 등 측정 미실시.
→ 개선: Gatling으로 SSE 부하 테스트, Grafana 대시보드로 정량화.
