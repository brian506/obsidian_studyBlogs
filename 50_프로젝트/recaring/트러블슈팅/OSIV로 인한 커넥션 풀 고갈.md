
## 개요

sse 데이터 통신 부하 테스트 도중, sse 연결 수가 DB 커넥션 풀 개수와 동일한 50개 연결에서 늘어나지 않고 계속 connection time out이 발생했다.

## OSIV?

> Open Session In View의 약자로 영속성 컨텍스트를 뷰 렌더링이 끝날 때까지 열어두는 것

### 동작 방식

1. **HTTP 요청 진입** : 클라이언트의 요청이 오면, Spring은 즉시 Hibernate Session(**영속성 컨텍스트**)을 연다.
2. **첫 DB접근(커넥션 획득)** : 요청을 처리하다가 서비스나 컨트롤러에서 최초로 DB 쿼리를 날리면, 열려 있던 session이 커넥션 풀로부터 물리적 DB 커넥션을 하나 가져온다.
3. **HTTP 응답 대기(커넥션 점유)** : 원래는 서비스 단에서 트랜잭션이 전부 수행되면 반납해야 하지만, 컨트롤러에서 지연 로딩 여부가 확정되지 않았기 때문에 **영속성 컨텍스트를 HTTP 응답이 끝날 때까지** 살려둔다.

### OSIV의 존재 이유

OSIV는 Thymeleaf, JSP 같은 서버 사이드 렌더링 시대에 등장한 패턴이다.
OSIV는 컨트롤러에서 지연 로딩을 사용하기 위해 존재한다.

**예시**
1. 서비스 단에서 `Member` 엔티티를 조회해서 컨트롤러에 넘긴다.
2. 서비스의 트랜잭션이 끝났다.
3. 컨트롤러가 응답을 주기 위해 `Member` 객체를 JSON으로 변환(직렬화)한다.
4. Jackson 라이브러리가 `member.getTeam().getName()` 을 호출해서 값을 추출한다. - **이것이 지연로딩**
5. **OSIV가 꺼져있으면** 영속성 컨텍스트가 닫혔기 때문에 에러 발생
6. **OSIV가 켜져있으면** HTTP응답이 끝날때까지 영속성 컨텍스트는 열려있으므로, DB로 쿼리를 날려 값을 받아온다.
	-  @Transaction 밖에서 엔티티의 연관관계를 참조하는 순간, 영속성 컨텍스트는 DB 커넥션을 다시 가져온다. 
		-> **트랜잭션 밖에서 일어나는 모든 DB 접근은 커넥션을 반환할 기준이 없어 Session이 닫힐 때까지 점유**된다.
		
# SSE에서의 트러블슈팅

일반 REST 요청에서는 OSIV가 큰 문제가 되지 않는다. 요청이 수 밀리초 안에 끝나기 때문에 커넥션을 잠깐 더 들고 있어도 금방 반환된다.

SSE는 **HTTP 응답이 완전히 종료되지 않은 채로 데이터를 흘려보내는 방식**이다. OSIV입장에서는 아직 요청이 끝나지 않았다고 판단해 **Session을 닫지 않는다.**

현재 나는 스트리밍해야 하는 GPS 값을 트랜잭션 밖에서(트랜잭션 경계가 없음) Look Aside 패턴으로 읽어와 캐시 미스가 발생했을 땐 DB에 가져온다.
- 여기서 DB에 접근하여 커넥션을 획득
	- SSE 스트림이 닫힐 때까지 반환 안됨	- 

## OSIV 가 원인임을 의심하게 한 원인들

### 1. 커넥션 풀 고갈 로그

```
Unable to acquire JDBC Connection
  HikariPool-1 - Connection is not available, request timed out after 30000ms
```

### 2. SSE 연결 수 = 커넥션 최대 크기

**의심되는 상황**
1. 그라파나에서 확인한 SSE 연결 숫자가 커넥션 최대 크기인 50과 동일한 상태로 늘어나지 않았다.
2. SSE는 실제 DB 쿼리를 하지 않는데, 커넥션을 차지하고 있었고 이를 놓지 않았다.
3. 일반 REST API 요청은 커넥션을 금방 반환하는데, SSE 요청만 계속 점유한다.

**DB를 안 쓰는 SSE 스트리밍 구간에서 커넥션이 잡혀 있다는 것**이 가장 의심하게 되는 요소이다.

### 3. 메서드 위에 @Transaction이 있는 경우(READ 쿼리)
- 쓰기 쿼리는 정상적으로 커넥션을 바로 반납한다.
- 하지만 읽기 쿼리는 트랜잭션 어노테이션을 명시하지 않아도 readOnly = true가 디폴트값이다.

## 해결

```yml
# application.yml
spring:
  jpa:
    open-in-view: false
```

지금 프로젝트 구조는 연관관계를 사용하지 않으므로 지연로딩을 고려하지 않아도 돼서 그냥 OSIV 설정을 꺼주면 된다.
