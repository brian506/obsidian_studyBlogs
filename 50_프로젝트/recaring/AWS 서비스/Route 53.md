반복적인 DNS 쿼리 이후에 해당 도메인의 IP주소를 반환해주는 AWS의 DNS 서버이다.
*도메인을 Route 53에서 사지 않더라도, 해당 도메인을 Route 53에서 할 수 있다.*

크게 2가지 역할을 한다.
1. Publlic DNS(도메인 -> IP주소 변환)
2. Private DNS(VPC 내부에서만 동작하는 DNS)
	- CloudMap이 사용

### Route 53 레코드 타입
| 타입        | 역할                                    | 예시                                  |
| --------- | ------------------------------------- | ----------------------------------- |
| **A**     | 도메인 → IPv4                            | `re-caring.com → 13.125.1.2`        |
| **AAAA**  | 도메인 → IPv6                            | `re-caring.com → 2001:db8::1`       |
| **CNAME** | 도메인 → 다른 도메인                          | `www.re-caring.com → re-caring.com` |
| **MX**    | 메일 서버 지정                              | 이메일 받을 때                            |
| **NS**    | 이 도메인 관리하는 네임서버                       | `re-caring.com → ns-123.awsdns.com` |
| **TXT**   | 텍스트 (도메인 소유 증명, SPF 등)                | `_acme-challenge → "abc123"`        |
| **Alias** | AWS 전용. ALB, CloudFront 등 AWS 리소스 가리킴 | A 레코드처럼 보이지만 IP 대신 AWS 리소스          |

### EC2에 Elastic IP를 쓰는 이유
EC2 인스터스를 재시작할 때마다 Public IP는 바뀐다. 
- 따라서 서버의 IP가 계속 바뀌게 되면 사용자의 접속이 차단되므로, 고정IP인 Elastic IP를 발급 받아 EC2에 연결해서 사용해야 한다.

### Route 53에서의 도메인 발급
도메인을 발급하기 위해서 해당 도메인에 IP주소를 연결한다. - **단일 서버일 때**
Route 53에서 DNS 쿼리 이후 IP 주소를 사용자에게 반환해줄때, 위와 같은 이유로 고정된 IP 주소를 반환해야 하므로 도메인에 Elastic IP주소를 연결한다.
- 이때 도메인 -> IPv4 인 **A 레코드**를 사용한다.

### 과금 
- EIP는 인스턴스에 붙어 있으면 무료, 안 붙어 있으면 시간당 비용이 발생한다.

### 만약 Nginx(EC2)가 아니라 ALB를 로드밸런서로 사용한다면?
- ALB는 EIP를 사용하지 않는다.
- ALB는 트래픽 양에 따라 내부적으로 노드를 늘리거나 줄이면서 **IP를 동적으로 계속 바꾼다.**
	- 따라서 Route 53에서 **Alias 레코드**를 사용한다.
	
**라우팅 방식**
1. Route 53 Alias 레코드 : 도메인 -> **ALB의 공유 DNS 이름** 연결
	- Route 53이 ALB의 DNS이름을 바라보고 있으면, ALB의 IP가 바뀌어도 Route 53이 알아서 바뀐 IP로 트래픽을 보내준다.
		- Route 53은 ALB의 IP가 바뀌는 것만 알아서 매칭
2. ALB -> 컨테이너 
	1. 컨테이너가 새로운 Private IP를 할당 받을 때마다, ALB에 연결된 **대상 그룹**에 적어둠.
	2. ALB의 대상 그룹에 있는 IP번호대로 트래픽 전송
