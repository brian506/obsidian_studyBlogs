# 스택·큐·덱

## 개념
- **Stack (LIFO)**: Last In First Out. 마지막에 넣은 게 먼저 나옴 (책 쌓기)
- **Queue (FIFO)**: First In First Out. 먼저 넣은 게 먼저 나옴 (줄 서기)
- **Deque (Double-Ended Queue)**: 양쪽 끝에서 삽입·삭제 모두 가능

## 핵심 연산과 복잡도 (모두 O(1))
| | Stack | Queue | Deque |
| --- | --- | --- | --- |
| 삽입 | push | offer | offerFirst / offerLast |
| 삭제 | pop | poll | pollFirst / pollLast |
| 조회 | peek | peek | peekFirst / peekLast |

## 활용 사례

### Stack
- **함수 호출 스택** (재귀 = 암묵적 스택)
- **수식 괄호 검사**, 후위 표기법 계산
- **DFS** (명시적 스택 또는 재귀)
- **Undo/Redo** 기능
- **브라우저 뒤로가기**

### Queue
- **BFS** (탐색)
- **CPU 스케줄링** (FCFS, RR의 Ready Queue)
- **메시지 큐** (Kafka, RabbitMQ — 분산 시스템 큐)
- **프로듀서-컨슈머** 패턴, 버퍼

### Deque
- **슬라이딩 윈도우 최댓값/최솟값** (Monotonic Deque)
- 양방향이 필요한 큐
- Stack과 Queue 모두로 활용 가능 → 사실상 표준 선택

## 변형
- **Priority Queue**: 우선순위 높은 것부터 → [[힙과 우선순위 큐]] 참고. 큐가 아닌 **힙**으로 구현
- **Circular Queue**: 고정 크기 배열을 원형으로 → 메모리 재사용 (네트워크 버퍼)
- **Blocking Queue**: 비어있으면 대기, 꽉 차면 대기 → 스레드 통신 (`LinkedBlockingQueue`)

## Java Collections 매핑
| 구조 | 추천 클래스 | 비고 |
| --- | --- | --- |
| Stack | `ArrayDeque` (`push`, `pop`) | `java.util.Stack`은 레거시(synchronized), 비추 |
| Queue | `ArrayDeque` 또는 `LinkedList` | ArrayDeque가 캐시 친화·빠름 |
| Deque | `ArrayDeque` | 표준 선택 |
| Priority Queue | `PriorityQueue` | 힙 기반 |
| Blocking Queue | `LinkedBlockingQueue`, `ArrayBlockingQueue` | 멀티스레드용 |

> **요점**: 대부분 `ArrayDeque` 하나면 됨. `java.util.Stack`은 피하고 `Queue` 인터페이스에 `ArrayDeque` 또는 `LinkedList`.

## 면접 빈출 질문
1. **스택과 큐의 차이?** → LIFO vs FIFO. 입출구의 방향
2. **재귀를 스택으로 바꿀 수 있나?** → 가능. 함수 호출 스택을 명시적으로 관리. 깊이 제한이나 성능 최적화 시 사용
3. **Java에서 Stack 클래스를 쓰지 말라는 이유?** → Vector 상속(synchronized) → 느림 + 일관성 깨짐. `ArrayDeque` 사용
4. **DFS와 BFS의 자료구조?** → DFS=스택(또는 재귀), BFS=큐
5. **Priority Queue는 큐인가?** → 이름은 큐지만 내부 구현은 **힙**. FIFO 아님. 우선순위 기준으로 dequeue

## 코드 패턴 (Java)
```java
// Stack 대신 Deque
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1); stack.pop();

// Queue
Queue<Integer> queue = new ArrayDeque<>();
queue.offer(1); queue.poll();

// Deque
Deque<Integer> dq = new ArrayDeque<>();
dq.offerFirst(1); dq.offerLast(2);
dq.pollFirst();  dq.pollLast();
```
