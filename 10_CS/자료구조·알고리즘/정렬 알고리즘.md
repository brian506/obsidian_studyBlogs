# 정렬 알고리즘

## 알아야 할 정렬 비교표

| 알고리즘 | 평균 | 최악 | 공간 | 안정성 | In-place | 비고 |
| --- | --- | --- | --- | --- | --- | --- |
| 버블 정렬 | O(N²) | O(N²) | O(1) | ✅ | ✅ | 교육용 |
| 삽입 정렬 | O(N²) | O(N²) | O(1) | ✅ | ✅ | 거의 정렬된 상태에선 빠름. **N 작을 때 우수** |
| 선택 정렬 | O(N²) | O(N²) | O(1) | ❌ | ✅ | swap 횟수 최소 |
| **병합 정렬** | **O(N log N)** | **O(N log N)** | O(N) | ✅ | ❌ | 분할 정복, 외부 정렬 |
| **퀵 정렬** | **O(N log N)** | O(N²) | O(log N) | ❌ | ✅ | 평균은 가장 빠름. 캐시 친화 |
| **힙 정렬** | O(N log N) | **O(N log N)** | O(1) | ❌ | ✅ | 최악 보장 + in-place |
| **Tim Sort** | O(N log N) | O(N log N) | O(N) | ✅ | ❌ | Python/Java 표준 (병합 + 삽입) |

> **안정 정렬**: 같은 값을 가진 원소들의 **상대 순서가 보존**되는 정렬
> **In-place**: 추가 메모리를 거의 안 씀 (O(1) 또는 O(log N))

## 비교 정렬의 하한
비교 기반 정렬은 어떤 알고리즘이든 **최소 O(N log N)** 이 필요 (정보 이론적 하한).
→ 더 빠르려면 비교 외의 정보를 활용해야 함 → 다음 항목.

## 비교 없는 정렬 (선형 시간)

### Counting Sort
- 값의 범위가 작을 때(K) → **O(N + K)**
- 안정 정렬, in-place 아님

### Radix Sort
- 자릿수 단위로 Counting Sort 반복 → **O(D × (N + K))**, D는 자릿수
- 정수·문자열에 적합

### Bucket Sort
- 값의 분포가 균등할 때 N개의 버킷으로 분배 후 각자 정렬 → 평균 O(N)

→ 면접에선 "비교 정렬의 한계와 선형 정렬의 조건" 정도만 답하면 충분

## 주요 알고리즘 핵심 아이디어

### 퀵 정렬
1. **피벗** 선택 (랜덤·중앙값 권장, 양 끝은 피하기)
2. 피벗 기준 작은 값 / 큰 값 분할
3. 두 부분 재귀

- 평균 O(N log N), **최악 O(N²)** — 피벗이 항상 최소/최대
- in-place, **불안정**
- 캐시 친화적이라 실측 성능 최고

### 병합 정렬
1. 절반으로 분할
2. 재귀로 각각 정렬
3. **두 정렬된 배열 병합**

- 항상 O(N log N), **안정**
- 추가 공간 O(N) — 단점
- 연결 리스트에 적합 (병합 시 추가 공간 거의 X)
- **외부 정렬**(메모리에 안 들어가는 큰 데이터)에 표준

### 힙 정렬
1. 배열을 힙으로 (O(N))
2. 루트(최댓값) pop을 N번

- 항상 O(N log N), in-place, **불안정**
- 평균 속도는 퀵보다 느림 (캐시 비친화)

### Tim Sort (Java 7+ Arrays.sort(Object[]), Collections.sort)
- **병합 + 삽입** 하이브리드
- 이미 정렬된 부분 구간(run)을 찾아 활용 → 거의 정렬된 데이터에 O(N)
- 안정 정렬

> ℹ️ Java `Arrays.sort(int[])` 같은 **primitive** 배열은 Dual-Pivot **퀵 정렬** 사용 (성능 우선, 안정성 불필요)
> 객체 배열·컬렉션은 **Tim Sort** 사용 (안정성 필요)

## 면접 빈출 질문
1. **퀵 정렬의 시간 복잡도와 최악 케이스?** → 평균 O(N log N), 최악 O(N²) — 피벗이 항상 최소/최대일 때
2. **퀵과 병합 비교?** → 평균 속도 비슷하지만 퀵이 캐시 친화로 실측 빠름 / 병합은 최악 보장 + 안정, 추가 메모리 필요
3. **안정 정렬이 왜 중요한가?** → 다중 키 정렬(예: 이름 정렬 후 점수 정렬) 시 1차 순서 유지
4. **N이 작을 때 어떤 정렬이 빠른가?** → **삽입 정렬** — 오버헤드가 작음 (Tim Sort도 작은 구간에선 삽입 사용)
5. **Java의 정렬 알고리즘은?** → primitive=Dual-Pivot QuickSort, Object/Collection=Tim Sort
6. **비교 정렬의 하한이 O(N log N)인 이유?** → 비교 결정 트리의 leaf가 N!개 → 트리 높이 ≥ log(N!) = O(N log N)
7. **선형 정렬이 가능한 조건?** → 비교를 안 하고 값의 분포·범위 정보를 활용 가능할 때 (Counting/Radix/Bucket)

## 코드 패턴 (Java)
```java
// 기본 정렬
Arrays.sort(arr);                              // 오름차순
Arrays.sort(arr, 0, n);                        // 부분 정렬
Arrays.sort(boxedArr, Collections.reverseOrder());

// 객체 정렬
list.sort(Comparator.comparingInt(Person::getAge));
list.sort(Comparator.comparingInt(Person::getAge)
                    .thenComparing(Person::getName));
```
